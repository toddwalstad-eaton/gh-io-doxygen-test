Detailed Description

Time-mapping conversion specifier class.

This class encapsulates the behavior of the string-form date/time conversion specification. The specifier string is a series of letters with the same meaning as those used in the C99/POSIX strftime() call. Available conversions are: "d", "H", "I", "j", "m", "M", "S", "u", "w", "y", and "Y". In addition, "q" and "Q" are available as extensions for specifying milliseconds. Each position in "date_spec" is represented as a byte in the resulting series of Modbus-TCP registers; if two identical conversions are used within the same register, e.g. _spec[4]==_spec[5]=='Y', the conversion result will be stored as a 16-bit integer occupying the whole register. Otherwise, the conversion result will be stored as an 8-bit integer occupying half of a register, which can result in truncation for some conversions ('Y', for example).
