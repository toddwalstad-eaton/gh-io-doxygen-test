Detailed Description

Derived Class to manage the "update" steps for the LinuxImage image_type.

There are two phases here:

  • the preliminary checks for new_version and target device, plus step count determination (adds 3 to the total number of steps)
  • updating the listed files in their given order, processing the "classic" types for updating the Linux files

bool LinuxImageTypeHandler::prelim_check_steps ( px::id_t  dev_id,
const CFwUtil::image_update_params_t &  update_params 
)
virtual

Method to do the preliminary validity checks and step count for this image type for the indicated device.

The new_version should be checked to see that it is allowed (at present, we are not refusing any), and the product_arg should be checked to make sure that the targeted device is of the correct BSP/product type for this update (eg, not the wrong product). The device ID should also be that of the publisher. If any of these checks fail, the ImageResultsOob should be updated with the failure reason and marked done, and this method will have a false return value. If the checks succeed, the total step count in the ImageResultsOob should be increased by 3 (a somewhat arbitrary choice that we have settled on).

Parameters
dev_idThe targeted device's CDS ID. Must be the Publisher (=1).
update_paramsThe entry from the ImageList file for this image type. Will use the new_version and product_arg to determine validity.
Returns
True if no problems seen to do the indicated update, else False if a fatal error is detected and the update must not proceed.

bool LinuxImageTypeHandler::do_update_steps ( px::id_t  dev_id,
const CFwUtil::image_update_params_t &  update_params 
)
virtual

Method to do the update steps for the given type_file_list for the indicated device (publisher).

Each of the listed files should be updated as indicated by its type, as has been done in the past in the fw_update.sh script. The ImageResultsOob will be updated for each of the 3 steps, and when done (with success or failure), the ImageResultsOob will be updated to list this device as a success or failure case. Also on successful completion, will toggle the partition file and call for a reboot of this device.

Parameters
dev_idThe targeted device's CDS ID. Must be the Publisher.
update_paramsThe entry from the ImageList file for this image type. Will use the type_file_list and process for all files listed there in the given order.
Returns
True if no problems doing update steps (the expected result), else False if a problem is encountered that means that this device's firmware image was not updated for this image type.
