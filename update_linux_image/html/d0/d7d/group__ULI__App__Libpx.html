update_linux_image relies on libpx to provide:

  • The CDS API interface, almost entirely through the CdsHandler class
  • The ASIO framework
  • Definition of px basics:
    • Data types like px::string and px::CValue
    • Enum definitions for types, device properties, units, etc
  • The Mercury protobuf (messages) interface for the data that update_linux_image needs from the CDS:
    • Out-of-Band (OOB) configuration for Linux Image Update app
  • Utility functions:
    • The command line "options" parsing support
    • The debug syslogging functions and macros
