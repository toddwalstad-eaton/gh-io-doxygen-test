The usage for this app is very similar to that for the other image update apps, because the iot_agent expects to have them all use the same action arguments + paramters.

Usage: update_linux_image <action> [<action_args>] <options> where actions are: "make_version_oob <dev_ID>" (Creates the Image Version OOB[s]) "check_version <dev_ID> <new_version> <image_type>" (Returns 0 and "OK" text if OK to upgrade to this version) "update <dev_ID> <image_type> <update_OOB> <collection_archive>" update_OOB: eg, 'ImageUpdate/xxxx' for xxxx = the command UUID or instance number for the Update to be tracked collection_archive: eg tar file; provide fully qualified path and options are: "-Dxx" (no space) optional argument to set the Debug level "-U <user>" Set the user (and group) to <user> "-f Force update even if product/board ID does not match" </endcode>
