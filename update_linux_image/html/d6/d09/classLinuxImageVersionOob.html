Detailed Description

Class to manage creating and populating the Image Version OOB for the Linux Kernel and Rootfs Image.

This OOB should be named "ImageVersion/xxxx/LinuxImage/vvvv" where the "xxxx" portion is the "canonical" Device Model (eg, "IQ250") to be tracked, and vvvv is the version of the current firmware image (without the final git hash code, if included, as for the Linux version). It is necessary to add the version to the tag to distinguish image information when devices of the same model have different versions of firmware. It should contain: In the list of CDS Device IDs, just the Publisher's ID One Image entry, consisting of: Image name "LinuxImage" Present Image version (ie, product-level firmware revision). This will be, for example, "RI7-181". Additional Version Detail about this Image version. This will be from the Publisher Device's software_version. A Repeated list of Component Details. These will list version information for components within the Image (eg, the PX Red Toolkit version). The Hardware version (eg, board revision) for this Image, from the Publisher Device's hardware_version. For the Enum of Upgrade Limitations, sets Normal and Reboot Required, and perhaps No Rollback. For the Enum field to indicate additional capabilities, will list just the Reboot command. For the name of the program to handle the Image Update process, will provide this app's name.

This class inherits most of its functionality from the generic ImageVersionOob, and extends that by implementing the handlers to provide specific details for the LinuxImage OOB.
