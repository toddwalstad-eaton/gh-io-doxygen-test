Test this OOB field update against its given "IF" criteria.

Apparently letters were expensive the day the "IF" class was named; "IF" is a protobuf sub-message in configsubscriber_cfg.proto that contains a number of criteria that must be satisfied "IF" the log or script that holds this IF instance is to be acted on.

Parameters
fieldName of the OOB field to be tested (Optional; may be empty)
if_The protobuf IF structure containing the criteria to be evaluated
oobThe full OOB object containing this field
notificationThe full CDS notification containing this OOB object.
Returns
True if all the If criteria is met.

Log the given message to the Audit log if its "IF" criteria are met.

If no "log" protobuf information is present (the common case), then this function will construct a standard message for the log.

Parameters
cfgItemThe configuration protobuf specific to the field of interest, containing the message to log, and (optional) "IF" criteria.
oobThe OOB object (containing the field of interest) which is being updated.
notificationThe full CDS notification containing this OOB object.
Returns
1 if the message was logged successfully, -1 if logging was attempted but failed (how can this happen??), or 0 if no message was logged (because the criteria were not met).
