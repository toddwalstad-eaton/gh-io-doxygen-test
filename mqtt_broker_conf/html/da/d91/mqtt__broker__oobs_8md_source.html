1 # MQTT Broker Out-of-Band (OOB) Objects
2 
3 The Mosquitto MQTT Broker is supported by three OOB objects:
4 
5  * **MqttBrokerSettings**
6  * **ACCESS_CTRL/MQTT**
7  * **ACCESS_CTRL/MQTTS**
8 
9 The MqttBrokerSettings OOB is configured using the `mqtt_broker_conf` program. See [mqtt_broker_conf.md](./mqtt_broker_conf.md) for more information about this program.
10 
11 
12 ## MqttBrokerSettings OOB
13 
14 The MQTT Broker requires a mosquitto.conf file to control its operation.
15 The purpose of the MqttBrokerSettings OOB is to define that input, allowing the UI to change the
16 settings in that OOB. The mqtt_broker_conf program is run before starting the Mosquitto mqtt_broker
17 (in [its initialization script](./rootfs/etc/init.d/mqtt_broker)), and this program parses
18 this OOB and uses the contents to generate the appropriate mosquitto.conf output.
19 
20 The configuration options for mosquitto.conf are very extensive, so rather than create an
21 extensive custom protobuf definition for the MqttBrokerSettings OOB, the design is to use
22 repeated GenericConfigFileEntry entries, as defined in [GenericTableEntry.proto](../libpx/tk_proto/hg/GenericTableEntry.proto).
23 
24 For the complete description of all the options, see the [mosquitto.conf man page](https://mosquitto.org/man/mosquitto-conf-5.html)
25 and [its TLS configuration](https://mosquitto.org/man/mosquitto-tls-7.html).
26 
27 For a fuller discussion of configuring TLS options for MQTT for Eaton products, see the Confluence page on
28 [Securing MQTT Broker with TLS](https://confluence-prod.tcc.etn.com/display/LNXTK/Securing+MQTT+Broker+with+TLS).
29 
30 The sections below describe how the GenericConfigFileEntry fields are used to support configuration
31 of the mosquitto.conf file.
32

34 ### Fields for the Configuration file
35 
36 The fields described here configure one entry in the mosquitto.conf, either as `<key_label> [key_options] <value>`
37 or else `<key_label> <complex_text>`.
38 
39 #### key_label
40 
41 The (usually unique) label that can be used to identify this configuration element or entry.
42 Used as the key in a configuration file, though some configuration items may have multiple entries.
43 Also serves as the "name" for this entry. Required.
44 
45 #### key_options
46 
47 Not used.
48 
49 #### value
50 
51 The chosen value for this table entry, when it can be specified this way. May commonly be just kept as a string,
52 even if it represents some numeric or boolean value to the system.
53 
54 #### complex_text
55 
56 Text for a complex entry, not conforming to the simple `<value>` style.

60 ### Fields for the Selection metadata
61 
62 The fields described here are used by the UI to offer appropriate choices to the user when choosing
63 MQTT options.
64 
65 #### value_choices
66 
67 A semicolon-separated text listing of the choices for the "value" field, which can be displayed to the user,
68 in the cases where there is a fixed set of choices (eg, "true;false").
69 
70 #### value_minimum and value_maximum
71 
72 The minimum and maximum permitted value for the "value" field,
73 when it is a numeric type with a limited range.
74 
75 #### option_choices
76 
77 Not used.
78 
79 ### Fields for UI guidance or hints
80 
81 The fields described here are used by the UI widgets to organize the configuration options
82 and present them appropriately, and offer some guidance to the user when configuring.
83 
84 #### tool_tip
85 
86 A short one-line description of this entry, suitable for display eg, as a UI tool-tip.
87 
88 #### description
89 
90 Longer, more descriptive explanation of this entry.
91 Can provide guidance to the user when configuring.
92 
93 #### category
94 
95 A simple tag that can be used to group "like" entries in this table.
96 
97 #### rank
98 
99 A simple numeric ranking for this entry, which can be used
100 by the UI to display it in the proper order with respect
101 to the other entries in the table or category.
102 
103 #### display_hint
104 
105 Indicates if this entry should normally be displayed, or else if it is disabled or hidden by the UI.
106 
107 
108 ### Fields for Developer guidance
109 
110 #### comments
111 
112 Optional developer comments about this entry.
113 May be useful to explain choices or dependencies.
114 Not intended for UI display.

196 ## ACCESS_CTRL/MQTT OOB
197 
198 For the OOB that controls the firewall settings for the MQTT Broker,
199 see [the writeup for all ACCESS_CTRL OOBs](../../documents/Network_OOB_Objects.md)
200 
201 By default, the MQTT Broker is disabled, and the default port assignment is the standard MQTT port 1883.
202 
203 ### Example 'ACCESS_CTRL/MQTTS'
204 
205 ```json
206  {
207  "toolkitStruct":
208  {
209  "access_ctrl":
210  {
211  "restricted":false,
212  "disabled":true,
213  "port":1883
214  },
215  "description":"Handles MQTT pub/sub requests, and access of MQTT data sources.",
216  "label":"MQTT Broker"
217  }
218  }
219 ```
