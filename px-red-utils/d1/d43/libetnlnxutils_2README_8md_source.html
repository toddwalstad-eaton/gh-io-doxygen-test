1 # libetnlnxutils Documentation
2 
3 `libetnlnxutils` is a library which enables execution of linux commands and shell scripts through CPP code.
4 
5 `libetnlnxutils` contains a single namespace `etn`.
6 The library can be can be distingushed into the following parts based on the functionality:
7 
8 ## 1. class `Exec`:
9 Functions of `Exec` class can be used to execute the shell scripts through CPP code.
10 It is inherits the class `ExecPrivate`.
11 
12 For example, if a shell script 'introduction.sh' is to be executed. Its contents are:
13 > $ cat introduction.sh
14 > #!/bin/sh
15 > echo "$1, who am I talking to?"
16 > read varname
17 > echo "It's nice to meet you $varname!"
18 
19 So the above script requires the greeting such as 'Hello' or 'Good morning' as an argument.
20 And, it requires name (varname) as the stdin input during runtime.
21 Suppose the script is present in path '/usr/bin/' on the device.
22 
23 To execute the above script using `libetnlnxutils`, there are two ways:
24 
25 #### a. Using the constructor:
26 _Constructors:_
27 - `Exec(const std::vector<std::string> arg, const std::string & input = "", uint32_t timeout = 0);`
28 - `Exec(const std::vector<std::string> arg, uint32_t timeout = 0);`
29 
30 _Usage:_
31 - `etn::Exec object( { "/usr/bin/introduction.sh", "Hello" }, "Jon Doe", 3);`
32 - `etn::Exec object( { "/usr/bin/introduction.sh", "Hello" }, 3);`
33 
34 Here,
35 - The first argument is a vector of the shell script path and the arguments required to be passed to the script.
36 - The second argument is the stdin input required by the script during run time (optional argument).
37 - The third argument is the timeout or time to wait for the script to execute before killing it forcefully.
38 
39 You can refer the `px-red-certmgr` module for more examples on usage.
40 
41 #### b. Using the function:
42 _Function definition:_
43 `int exec(const std::vector<std::string> arg, const std::string & input, uint32_t timeout, std::stringstream * stdout, std::stringstream * stderr);`
44 
45 _Usage:_
46 `std::stringstream out, err;`
47 `int rv = etn::exec( { "/usr/bin/introduction.sh", "Hello" }, "Jon Doe", 3, &out, &err);`
48 
49 Here,
50 The first three arguments are same as those with the constructor.
51 - Vector of the shell script path and the arguments required to be passed to the script.
52 - Stdin input required by the script during run time.
53 - Timeout or time to wait for the script to execute before killing it forcefully.
54 - The additional arguments 'out' and 'err' are output arguments which store the stdout and stderr messages output by the script during or after execution.
55 - The return value of the function is the integer value of status of the script whether success or failure.
56 
57 You can refer the `px-red-time-change-notifier` module for more examples on usage.
58 
59 
60 ## 2. class `CLinuxUtil`:
61 Functions of this class can be used to execute a command line in shell but through CPP code.
62 A single shell command or multiple commands in a single line can be executed using this class.
63 Functions from this class also provide logging support during the execution of the commands.
64 
65 For example, command line to be executed is:
66 `cdscmd -gd && cdscmd -gu`
67 
68 Above command(s) can be executed as:
69 _Function definition:_
70 `bool shell_cmd(const char* label, const char* cmd, bool leave_newline, int& cmd_status, std::vector<std::string>& outlines)`
71 
72 _Usage:_
73 `bool rv = shell_cmd("cds data", "cdscmd -gd && cdscmd -gu", true, status, outlines);`
74 
75 Here,
76 - The first argument is a label or tag describing the command line.
77 - The second argument is the actual command(s) to be executed.
78 - The third argument is a flag indicating whether to strip the new line character at the end of the output.
79 - The fourth argument is the integer value of the command status whether success or failure.
80 - The last argument is the combined stdout and stderr output of the command(s).
81 - The return value of the function is a boolean flag indicating success or failure.
82 
83 You can refer `px-toolkit/etn_source/iot_agent` for more examples on usage.
