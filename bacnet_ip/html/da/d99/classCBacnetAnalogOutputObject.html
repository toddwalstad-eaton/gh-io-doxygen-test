Normal constructor.

When constructing a CBacnetAnalogOutputObject, the use_cmd option determines whether to write to a command or a channel. For use_cmd = true, a command is written with the parameter "value" set to the BACnet client's value entered, and the parameter "priority" set to the BACnet client's priority entered, values 1-16. If your application doesn't have special meaning for priority index, construct this object with use_cmd = false.

Initializing the present-value: since the BACnet present-value should always be equal to the CDS value of the corresponding channel (when use_cmd = false), but sometimes is separately managed by a Device Handler, we must find a way to initialize this BinaryOutput to correspond to that value. By default, we start with an all-NULL priority array, and the present-value must equal the relinquish-default; we will defer to the shared mapping library to provide the value of relinquish-default to use here. Unfortunately, we can't get the Channel value in this constructor because it is invoked inside a CDS channel settings callback, so we defer initializing the relinquish-default until first use.

Parameters
objectTypeThe type (class) of BACnet Object this is. Should always be either OBJECT_ANALOG_INPUT or OBJECT_ANALOG_VALUE.
objectInstanceThe unique Object_Instance number for this Object.
objectNameThe name to give to this Point (eg, MCL Channel name).
bIsWritableTrue if the Present_Value of this Point is Writable.
cdsValueTypeThe CDS type of the Present_Value of this Point.
parentObjPointer to our parent Device Object.
bip_corePointer to our core class, which gives us access to CDS and application data.
channel_idThe CDS ID for the Channel for this Point instance, as it normally is based on a Channel (else, it is set to 0).
use_cmdTrue to use device command for writing; the object name is also the command name.
dev_idIf use_cmd is true, run commands against this device ID

void CBacnetAnalogOutputObject::update_cov_value ( const px::CValue &  updated_value)
virtual

Function to accept an updated value for the channel, and determine by its own rules whether this change constitutes a COV update or not.

For AnalogOutput, it might be unusual to receive an update this way from polled data.

This implementation will compare the new value to the last COV published value, and if it exceeds the COV-Increment, will set the Has Update flag to mark the need to send a COV notification.

Parameters
updated_valueThe new value to be tested against the last published for COV.
