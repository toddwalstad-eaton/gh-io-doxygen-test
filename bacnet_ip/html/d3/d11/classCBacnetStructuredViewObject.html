Detailed Description

The Structured View is a container object in BACnet, which organizes some of the parent device's channel objects, from the parent's flat list, into one level of hierarchy under this view.

We will use the SV primarily for subdevice data, such as that found in Branch Circuit, KYZ, or Outlet subdevices.

Each SV is assigned an Object ID which is its instance number (eg, 1-100) * _STRUCT_VIEW_SCALAR. Then each mapped channel under this SV will take its MCL mapping assignment and add it to this SV Object ID. For example, the first SV would be assigned Object ID 40,000, and its channels would be Object IDs of 40,001 to about 54,000. The second SV would be 80,000 and its channels 80,001 to 94,000.

Note: *** For the present release, *** we will not add this to the parent devices's object map or expose any SV properties via BACnet. We are just using this class as a container to organize the channel data that will ultimately be part of the legitimate BACnet SV object.

Utility function to return the calculated Object ID to use for a new StructuredView object, given its instance number.

Will try to widely space these, though if the total number of subdevices exceeds _MAX_STRUCT_VIEWS, it will have to reduce the spacing and risk collisions of the BACnet objects under the subdevices.

Parameters

num_sub_devicesThe total number of subdevices that the parent will have to map to SVs.
Returns
The calculated Object ID to use for a new SV, or 0 in case of failure.

const uint32_t CBacnetStructuredViewObject::_STRUCT_VIEW_SCALAR = 40000
static

The scalar for safely numbering Structured View Objects.

Multiply the instance number by this value to get the Object ID for a new Structured View. Currently set to 40,000 to be big enough to be beyond all the MCL mapped channel Object IDs (circa 14,000), but still leave enough Object ID space to enumerate all the SVs we might see (so, up to 1023 SVs in one device).

const uint32_t CBacnetStructuredViewObject::_MAX_STRUCT_VIEWS = ( 0x400000 / _STRUCT_VIEW_SCALAR )
static

The maximum number of SVs that we can enumerate for one device with our choice of _STRUCT_VIEW_SCALAR.

Currently is 2**22 / 40,000 = 104.8576 or 104. Should be enough for most BCMs (eg, 42 points *2), but bigger systems will need to risk collisions with narrower spacing.

const uint32_t CBacnetStructuredViewObject::_NARROWER_SCALAR = 10000
static

A narrower scalar if the number of SVs exceeds _MAX_STRUCT_VIEWS (though it cannot exceed 256 since uint8_t is used in calculate_object_id() here).

Currently set to 10,000 (1/4 of _STRUCT_VIEW_SCALAR). Use of this spacing interval does risk collisions for the BACnet objects located within this SV, since it is < 14,000.
