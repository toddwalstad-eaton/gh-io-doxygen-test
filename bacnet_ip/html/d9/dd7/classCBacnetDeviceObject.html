Detailed Description

The class that provides Device data, mapped from the CDS data to BACnet.

This class uses both the real-time and static data from px::CDevice and the gateway or virtual device OOB data from the CDS to map to BACnet Device properties and also drives the mapping of CDS Channel data to BACnet "Point" objects.

void CBacnetDeviceObject::initialize_device_objects ( )

Initializes the default device objects (from a built-in list) and then the device's objects which can be created from its channel data.

Processes the list of default objects for any Device and adds the ones for which we have the data. Currently, adds asset_id, serial, and hardware_version as CharacterString objects.

Then processes the channel_settings array and tries to lookup by canonical channel name, and create any objects found in the MCL-to-BACnet map.

Add a Structured View object for a subdevice to our map, after calculating its Object ID.

Will first determine the instance number for this subdevice (from its network_address) and use this to determine an Object ID for this SV, produced by calculate_object_id(). A Subdevice doesn't get its own real BACnet Devices object, just a Structured View object under the parent BACnet Device object.

If a SV already exists in the map for this Object ID, nothing will be done.

Commands a device reinitialization, to a given state.

uint32_t CBacnetDeviceObject::Device_Object_List_Count ( BACNET_OBJECT_TYPE  objectType)

Get the total count of objects of desired type supported by this Device Object.

There are 3 ways to use this function, depending on the objectType value:

  1. The classic Device Object List count, for MAX_BACNET_OBJECT_TYPE
  2. The Device's Structured View List count, for OBJECT_STRUCTURED_VIEW
  3. A new mode of counting just objects of the given objectType, though it is not obvious if this mode has any actual uses other than for StructuredViews.
Note
Since many network clients depend on the Device Object List for discovery, it must be consistent!
Parameters
objectTypeThe desired type (class) of BACnet Object to be counted; if a specific value is given (eg, OBJECT_STRUCTURED_VIEW), only that type will be counted; use MAX_BACNET_OBJECT_TYPE to mean "All (but StructuredView)"
Returns
The count of objects of the desired type, or else, for the "All" case, all supported Object types, including the Device Object but excluding the Structured View objects.

This standard Device Object Database_Revision property is a logical revision number for the device's database.
