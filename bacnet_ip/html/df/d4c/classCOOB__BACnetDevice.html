Detailed Description

Class to match and manage the px::proto::BACnetDevice class.

This class holds the BACnet Database data for one device, including any persisted properties.

Normally initialized from existing OOB data, but can fill out a new OOB structure with this Device's "configuration" data.

When creating a BACnet Device Object initially, it is necessary to have all the data it will need read into this class from the OOB data. Similarly, when writing out the Database OOB from this class' data, all the fields must be properly set here to be persisted correctly. Consequently, the members of this class follow the px::proto::BACnetDevice protobuf fields fairly well.

uint8_t COOB_BACnetDevice::mac_address[BIP_MAC_LEN]

The BACnet "MAC" address for this Device, as a byte array.

Formed as an IPv4 address plus the 0xBAC0 port number (normally). For the routed devices, to fairly well ensure a unique MAC template, we will turn around the gateway's IPv4 address and set the 4th byte to the "next" device instance #. So for gateway address and port 10.222.48.131:0xBAC0, the template for the routed MAC addresses is 131.48.222.xx.0xBA.0xC0 where xx will be this device's device instance #.

uint32_t COOB_BACnetDevice::_apdu_timeout

The persistent value of the device's apdu_timeout property.

This is the timeout (in msecs) for unacknowledged messages that require acknowledgement, after which they will be re-transmitted.

Though persistable on a per-device basis, it only applies to the main or gateway device. Per the updated spec, this should default to 10,000 since it is writable.
