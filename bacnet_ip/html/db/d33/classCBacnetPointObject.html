Detailed Description

Abstract Base Class for several BACnet "Point" measures: These types have many common properties that can be implemented in this one common class, with the outlying properties handled by derived class types.

Even the outlying properties are often shared by two or more of these types. The supported properties for each class are provided by local BacPropEntry_t arrays to differentiate among the types:

  1. OBJECT_ANALOG_INPUT
  2. OBJECT_ANALOG_VALUE
  3. OBJECT_BINARY_INPUT
  4. OBJECT_BINARY_VALUE
  5. OBJECT_MULTI_STATE_INPUT
  6. OBJECT_MULTI_STATE_VALUE
  7. OBJECT_CHARACTERSTRING_VALUE
  8. OBJECT_INTEGER_VALUE
  9. OBJECT_DATETIME_VALUE

int CBacnetPointObject::Object_Read_Property ( BACNET_READ_PROPERTY_DATA *  rp_data)
virtual

Reads one property for this object type of a given instance.

Extends the base class' implementation by incorporating properties common to the Point-type Objects.

Manages the property by calling handlers in this order until one handles it:

  1. Calls ObjectReadPropertyFromList() to let the base CBacnetObject class handle the simple properties common to all objects
  2. Prefetches a value from the CDS if this property has a field name hint
  3. Calls handle_read_property() so the derived class can handle its special properties
  4. Else, handles the simple properties common to the CBacnetPointObject types.
Parameters
rp_dataPointer to the BACnet_Read_Property_Data structure, which is packed with the information from the RP request.
Returns
The length of the apdu encoded, or 0 for null response, or -1 for error or -2 for abort message.
