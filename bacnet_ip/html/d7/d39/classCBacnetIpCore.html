Detailed Description

BACnet/IP application's Main loop class in the ASIO framework.

This class is used to handle overall program set-up/shut-down, initialize semi-global objects, and perform the main processing loop.

It implements the functionality defined in the base CServerCore class, and adds application-specific functions and data structures for BIP.

It implements the CTimedObject class for a sub-second timebase timer.

bool CBacnetIpCore::config ( )
virtual

This function sets up the configuration for the main loop().

It:

  • Creates the Manager for the ASIO framework
  • Creates IO Actors for the Signal Handling and Watchdog, and add them to ASIO
  • Creates an Actor to manage the CDS and add it to ASIO
  • gets the settings for BACnet/IP from the OOB object

Split from CBacnetIpCore constructor to facilitate Unit Tests. Done as virtual so the test class can override it.

Returns
True if the configuration steps succeeded, else False if BACnet/IP is not enabled or other fatal problem was detected.
Exceptions
init_errIf there was a problem initializing the main loop object. This can happen for any number of reasons; more specific information is printed at lower levels, and this error is propagated back to indicate that the caller should exit.

Reimplemented in CMockConfigCore.

void CBacnetIpCore::execute ( )

Execute the main loop.

Implements the virtual method from CServerCore.

This performs the "real" (steady-state) work of the server. This call will block indefinitely, and only return when the server has encountered some sort of non-recoverable error, or has been shut down by a user signal.

The operation is

  • Create the appropriate UDP (BACnet/IP listener) Actor
    • Add this IO Actor to ASIO and Kick it off
  • Start ASIO running
  • If the px::asio::CIoManager::run() loop should ever exit,
    • Quit the Manager and exit this main loop.
