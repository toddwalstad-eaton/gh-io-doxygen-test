snmpd_conf program

The snmpd_conf program is Eaton code to transform the input from the SnmpDataSettings, SnmpMibs, and SnmpUserSettings OOBs (Out Of Band) objects into an snmpd.conf file to control the operation of the Net-SNMP snmpd daemon. If any SNMPv3 users are configured (as encouraged now, in place of the insecure v1 community strings), the snmpd_conf program also reads in their SnmpV3UsmUser/<username> OOBs and stores their (hashed) information in the semi-hidden /flash/app/snmpd.conf file.

The purpose of the SnmpDataSettings, SnmpUserSettings, and SnmpV3UsmUser/<username> OOBs are to define that input, allowing the UI to change the settings in those OOBs. The snmpd_conf program is run before starting the Net-SNMP snmpd daemon (in its initialization script), and this program parses those OOBs and uses the contents to generate the appropriate snmpd.conf output.

The one rather unusual function of snmpd_conf is to arrange to capture and save the configuration for new SNMP v3 Users; this is a four-step process:

  1. The UI makes a new entry in the SnmpUserSettings OOB's "v3_users" section.
  2. snmpd_conf is run to generate an snmpd.conf file with a partial entry for this v3 user.
  3. snmpd is started; it processes the snmpd.conf file, and writes out an entry for this user in the semi-hidden /flash/app/snmpd.conf file, with the passphrase hashed.
  4. snmpd_conf is run again to capture the hashed entry contents and save them to SnmpV3UsmUser/<username>.

For the complete description of all the snmpd.conf options, see the snmpd.conf man page.

Information about running the Net-SNMP snmpd daemon can be found on the snmpd man page, and the other programs and libraries in the Net-SNMP suite are also described near there.

snmpd_conf Usage

The usage and command line arguments for snmpd_conf are:

``` Usage: snmpd_conf [-D <verbosity>] [-Ns] [-h] [-hv] [-p <dir>] [-s] [-up <dir>] [-v]

-D : Set the application's debugging verbosity level -Ns : Don't direct log messages to syslogd -h : Print usage information to stderr and exit -hv : Print detailed/verbose usage information to stderr and exit -p : Directory path to which snmpd.conf is to be written. -s : Update user OOB objects from config files -up : Directory path to which snmpd.conf is to be updated with createUser. -v : Print version information to stderr and exit ```

The arguments used in a particular target can be found with the showcfg argument to the /etc/init.d/snmpd script:

``` snmpd_conf -p /etc -up /initlo/data/app ```
