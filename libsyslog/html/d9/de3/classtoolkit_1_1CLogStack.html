Detailed Description

This class is used to keep a "function stack" for logging purposes.

This is optimized for multi-threaded apps, since if none of the threads have a verbosity level that's high enough to output function entry/exit points, it doesn't waste time looking up the thread-specific verbosity level. Without this optimization, the app runs MUCH slower on development builds, even when nothing is being output.

Constructor & Destructor Documentation

toolkit::CLogStack::CLogStack ( const char *  func)

Constructor.

Upon construction, the object outputs the name of the function being entered (if the app's verbosity level is high enough). It also increments the global indentation level.

Each app can define the __DBG_FUNC_VERBOSITY it wants, normally in each app's log.hpp file. This is usually set to a pretty high (very verbose) debug level, eg 10.

Parameters
funcThe (pretty) name of the function being entered

void toolkit::CLogStack::set_max_verbosity ( unsigned int  new_max)
static

Set the verbosity ceiling for all threads to the new value.

This is useful for quickly skipping over unnecessary thread-specific pointer dereferences to check the thread's verbosity level, when it's known that none of the threads have a high enough verbosity to output anything.

Parameters
new_maxNew value to use as max verbosity for all threads.
