Detailed Description

Simple class to manage the { } JSON message for Audit logging.

Holds the fields that the applications can load to fill in the required and optional text fields for this message.

The resulting message will have the format (if every field were filled in) of :{"msg": "The explanation here", "action": "access", "status": "failure", "auid": "user", "host": "RA6NCWHP2002475", "ipv4": "10.222.41.124", "appname": "configsubscriber" }

This conforms to the format proposed by Lumberjack, based on CEE ideas: https://fedorahosted.org/lumberjack/wiki/FieldList#UnifiedFieldList

Allows us to validate the input and control the order of output. This class can be extended as we add support for new fields.

Since this class is holding pointers to external character arrays, it should exist temporarily and be used quickly before they become invalid.
