2 
3 This is the root project folder for the MQTT Device Handler and related components.
4 Note: The MQTT Device Handler is not a standalone service, and requires Adopter Project MQTT Device EDS file(s) in order to function properly.

20 ## How to enable the MQTT Device Handler in your product
21 Follow steps 1 and 2 below to enable building and running the MQTT Broker and MQTT Device Handler applications.
22 
23 ### Step 1: Enable building Mosquitto MQTT Broker, MQTT Broker Config, and MQTT Device Handler
24 
25 Edit your project root `config` file and add the following entries. Rebuild your project.
26 
27  ###############################
28  ## Eaton Software
29  ###############################
30 
31  # Enable MQTT Broker, Library, and associated components
32  CONFIG_EATON_MQTT_BROKER?=y
33  export CONFIG_EATON_MQTT_BROKER
34 
35  # Enable MQTT Library, Device Handler, and associated components
36  CONFIG_EATON_MQTT_DEV_HANDLER?=y
37  export CONFIG_EATON_MQTT_DEV_HANDLER
38 
39 ### Step 2: Enable MQTT Broker Config service
40 
41 #### Edit the project rc.conf.product file
42 Open the project `rootfs/etc/rc.conf.product` file and add the following lines to enable the MQTT Broker Config service.
43 
44  # Enable the MQTT Broker
45  mqtt_broker_enable=y
46 
47 #### Edit the project rc.platform.product file
48 Open the project `rootfs/etc/rc.platform.product` file and add system configuration references to the product specific MQTT Product EDS file, and the location of the product specific MQTT Device EDS file(s). Examples below:
49 
50 **Example PX-Red-BBB (product-ref1):**
51 
52  # Location of MQTT Device EDS files to be used by the MQTT Device Handler
53  CFG_MQTT_DEVICE_EDS_DIR="/eds/protocol/mqtt"
54 
55 ### Step 3: Add MQTT Device EDS File(s) to your Project
56 See the product-ref1 Adopter project as an example of adding MQTT Device EDS files to the build.
57 [Add MQTT Device EDS files to your project](https://bitbucket-prod.tcc.etn.com/projects/LNXTK/repos/product-ref1/eds/mqtt)
58 [Makefile to copy EDS files to build output diretcory](https://bitbucket-prod.tcc.etn.com/projects/LNXTK/repos/product-ref1/eds/mqtt/Makefile)
59 [Add 'eds' and 'eds_source' targets to project Makefile](https://bitbucket-prod.tcc.etn.com/projects/LNXTK/repos/product-ref1/Makefile)

61 ## MQTT Pub Test Command Line Examples
62 The `mqtt_pub_test` utility reads an MQTT Device EDS file and publishes channel test data to the MQTT broker. Only channels with a `test` object will have data published to the broker.
63 
64 * Run `mqtt_pub_test` once with debug level 5, unique Device Id `56781234`, and MQTT test device file `/eds/protocol/mqtt/test/device/EDS_MQTT-Test.json`.
65 ```
66  ~ # mqtt_pub_test --loop-count 1 -d -D 5 -I 56781234 -f /eds/protocol/mqtt/test/device/EDS_MQTT-Test.json
67 ```
68 
69 * Run `mqtt_pub_test` once with unique Device Id `56781234`, and MQTT test device file `/eds/protocol/mqtt/test/device/EDS_MQTT-Test.json`.
70 ```
71  ~ # mqtt_pub_test --loop-count 1 -I 56781234 -f /eds/protocol/mqtt/test/device/EDS_MQTT-Test.json
72 ```
73 
74 * Loop `mqtt_pub_test` continuously with a loop wait period of 1000ms, QoS level of 2, retain messages, unique Device Id `56781234`, and MQTT test device file `/eds/protocol/mqtt/test/device/EDS_MQTT-Test.json`.
75 Use CTRL-C to stop the program.
76 ```
77  ~ # mqtt_pub_test --loop-delay 1000 -q 2 -r -I 56781234 -f /eds/protocol/mqtt/test/device/EDS_MQTT-Test.json
78 ```
79 
80 * **Pro Gateway Project only**: Loop `mqtt_pub_test` continuously with a loop wait period of 1000ms, QoS level of 2, retain messages, Device serial # `12345678`, and MQTT device file `/eds/protocol/mqtt/device/EDS_GameOn400.json`.
81 Use CTRL-C to stop the program.
82 ```
83  ~ # mqtt_pub_test --loop-delay 1000 -q 2 -r -I 12345678 -f /eds/protocol/mqtt/device/EDS_GameOn400.json
84 ```

86 ## Using MQTT Device Handler to check MQTT EDS files for validty
87 `mqtt_device_handler` can be used to check MQTT Device EDS files for correctness.
88 
89 * Check MQTT Device EDS in a directory.
90 ```
91  ~ # /bin/mqtt_device_handler -gve /eds/protocol/mqtt -pv
92  ~ # /bin/mqtt_device_handler -gve /eds/protocol/mqtt/test/px-red/eds -pv
93 ```
