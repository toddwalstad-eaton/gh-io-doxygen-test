2 ============
3 
4 
5  **Utility to capture and replay select CDS data in a JSON format.** <br>
6 This tool is useful for: <br>
7  - backup and restore of configuration
8  - cloning of select data from system to system
9  - aiding in KVM testing by providing a full snapshot of a real system to start with
10 
11  The program has two basic operations:<br>
12  **1) Capture** <br>
13  Captures the selected (or all) CDS data and outputs to a file it in pretty-printed JSON format.
14 
15  **2) Replay** <br>
16  Reads the JSON file from a capture and replays (restores) the selected (or all) data to the CDS data.
17 <br><br>
18 
19  The **types of CDS data** that can be captured or replayed are (showing the tag names used on the command line): <br>
20  - *config*: Out-of-Band (OOB) objects (ie, configuration data)
21  - *users*: Users and Roles data
22  - *devices*: Device objects
23  - *states*: The Device or Channel states
24  - *channels*: The channel data found under each device
25  - *mcl*: The channel settings (descriptive parameters) from the Master-Channel List (MCL) - <i>not presently implemented</i>
26  - *values*: Includes the Channel Values with the other Channel parameters.

94  -i :
95  -include : [config,users,devices,states,channels,mcl,values] Specify the
96  include options for capture or replay operation.
97  Without '-i' option capture operation will include OOBs and user
98  management data where as replay operation will replay everything in

104  <devices>: Device objects in the capture.
105  <states>: All the Device or Channel states in a replay.
106  <channels>: Capture the list of channels under each device.
107  <mcl>: Capture the MCL-based channel settings information.
108  <values>: Channel Values with the other Channel data.
109  (no default)
