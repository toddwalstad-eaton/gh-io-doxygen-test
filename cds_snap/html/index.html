cds_snap Documentation

Utility to capture and replay select CDS data in a JSON format.
This tool is useful for:

  • backup and restore of configuration
  • cloning of select data from system to system
  • aiding in KVM testing by providing a full snapshot of a real system to start with

The program has two basic operations:
1) Capture
Captures the selected (or all) CDS data and outputs to a file it in pretty-printed JSON format.

2) Replay
Reads the JSON file from a capture and replays (restores) the selected (or all) data to the CDS data.

The types of CDS data that can be captured or replayed are (showing the tag names used on the command line):

  • config: Out-of-Band (OOB) objects (ie, configuration data)
  • users: Users and Roles data
  • devices: Device objects
  • states: The Device or Channel states
  • channels: The channel data found under each device
  • mcl: The channel settings (descriptive parameters) from the Master-Channel List (MCL) - not presently implemented
  • values: Includes the Channel Values with the other Channel parameters.

The command line usage illustrates the operation options for this program.

First, the basic (terse) usage information (obtained with the -? or -h options):

~ $ cds_snap -h

Usage: cds_snap [-V] [-c <file>] [-h] [-hv] [-i [config[,mcl,values]]] [-i config[,users][,devices[,states],channels[,mcl,values]]] [-iu] [-pp] [-pw <password>] [-r <file>] [-tok <token_string>] [-usr <name>] [-v] [-vv] [-wt <wait_time>]

-V : Print version information to stderr and exit -c : [<file>] Specify output filename. -h : Print usage information to stderr and exit -hv : Print detailed/verbose usage information to stderr and exit -i : [config,users,devices,states,channels,mcl,values] Specify the include options for capture or replay operation. -iu : Enables including unique data items for replay operation. -pp : Print prettier output. -pw : [<password>] Log into CDS using this password and username from '-usr'. -r : [<file>] Specify input filename. -tok : [<token_string>] Token for existing CDS session. -usr : [<name>] Username requesting CDS commands. -v : Print verbose output. -vv : Print even more verbose output. -wt : [<wait_time>] Try connecting to the CDS for at least <wait_time> seconds,if the CDS connection fails.

Secondly, the full (verbose help) usage information (obtained with the -hv option): 

~ $ cds_snap -hv  

Usage: cds_snap
                [-V] [-c <file>] [-h] [-hv] [-i [config[,mcl,values]]]
                [-i config[,users][,devices[,states],channels[,mcl,values]]]
                [-iu] [-pp] [-pw <password>] [-r <file>] [-tok <token_string>]
                [-usr <name>] [-v] [-vv] [-wt <wait_time>]

-V :
-version : Print version information to stderr and exit
           This option prints usage information, including the Toolkit version,
           build date, and revision number.  However, be aware that version
           information is only updated when this particular file is changed, so
           this information is not necessarily up to date - if in doubt, you
           should compare against a known firmware version.

-c :
-capture : [<file>] Specify output filename.
           Specify CDS Snapshot output filename (JSON format).
           (no default)

-h :
-help : Print usage information to stderr and exit
        This prints general usage information, including a program synopsis and
        general description for each option.

-hv :
-usage : Print detailed/verbose usage information to stderr and exit
         This prints usage information similar to that shown for "--help";
         however, extra details are shown for this flag, including a more
         in-depth discussion of each option and displaying default values.

-i :
-include : [config,users,devices,states,channels,mcl,values]    Specify the
           include options for capture or replay operation.
           Without '-i' option capture operation will include OOBs and user
           management data where as replay operation will replay everything in
           the given file except the unique data.Capture and Replay type
           options are:
           <config>: This option will select OOBs for capture or replay.
           'values' will be selected for capture operation only.
           <users>: Users and Roles data.
           <devices>: Device objects in the capture.
           <states>: All the Device or Channel states in a replay.
           <channels>: Capture the list of channels under each device.
           <mcl>:  Capture the MCL-based channel settings information.
           <values>: Channel Values with the other Channel data.
           (no default)

-iu :
-include_unique : Enables including unique data items for replay operation.
                  This flag enables including unique data items during replay
                  operation.
                  This has no effect on capture operation. By default capture
                  collects all data.
                  (default: false)

-pp :
-prettyprint : Print prettier output.
               Make "prettier" JSON output when used. Can be added to most
               commands that print JSON output.
               (default: false)

-pw :
-password : [<password>] Log into CDS using this password and username from
            '-usr'.
            Specify password for CDS login.
            (no default)

-r :
-replay : [<file>] Specify input filename.
          Specify CDS Snapshot input filename (JSON format).
          (no default)

-tok :
-token : [<token_string>] Token for existing CDS session.
         Specify a token to use an existing CDS login session.
         (no default)

-usr :
-username : [<name>] Username requesting CDS commands.
            Specify Username requesting CDS commands.
            (no default)

-v :
-verbose : Print verbose output.
           Print verbose output.
           (default: false)

-vv :
-veryverbose : Print even more verbose output.
               Print even more verbose output.
               (default: false)

-wt :
-wait_time : [<wait_time>] Try connecting to the CDS for at least <wait_time>
             seconds,if the CDS connection fails.
             Try connecting to the CDS for at least <wait_time> seconds, if the
             CDS connection fails.The action will not be run if a CDS
             connection cannot be made.
             (default: 2)
