1 # PTP Daemon (ptpd)
2 PX-Red uses a third party vendor source package LinuxPTP-3.1 to synchronize local PTP hardware clock or system clock with PTP master instance.
3 
4 At startup ,
5 - It reads the PTP configuration parameter from the 'PtpConfig' OOB and update the 'ptp4l' config file located at '/etc/ptp_default.cfg'
6 - Later, it starts the 'ptp4l' and 'phc2sys' service ('phc2ys won't spawned in case of Software Timestamping mode ').
7 - Then it spawns 'pmc'-PTP Management client utility to poll few status parameters at configurable poll interval and update it to 'PtpStatus' OOB.
8 
9 It supports both hardware and software timestamping slave mode with build time configurable parameter 'CONFIG_ENABLE_HW_TIMESTAMP_PTPD'
10 
11 The default is hardware timestamping mode.
12 
13 Hardware timestamp mode:
14 - "ptp4l" instance is spawned with hardware timestamping flag, which sync the local PTP hardware clock with PTP master's clock.
15 
16 - "phc2sys" is also spawned with "ptp4l", which sync the local system clock with local PTP hardware clock.
17 
18 Software timestamp mode:
19 - "ptp4l" instance is spawned with software timestamping flag, which sync the local system clock with master's clock.
20 
21 - "phc2sys" won't be spawned in this mode, as ptp4l itself sync the local system clock time with PTP master's clock.

34 # PTP Daemon Configuration and Status Out-of-Band (OOB) Objects
35 
36 The following PTP OOB objects support the configuration and status of the PTP daemon `ptpd`:
37 * PtpConfig - PTP configuration.
38 * PtpStatus - PTP status.

40 ## PtpConfig OOB
41 
42 Configurable fields for "ptp4l" service
43 
44 ### Fields
45 
46 #### enable
47 
48 Enable/Disable PTPD service.
49 
50 The default is false.
51 
52 ### network_transport
53 
54 Select the network transport. Possible values are:
55 
56 - "UDPv4"
57 - "UDPv6"
58 - "L2" - Raw Ethernet
59 
60 The default is L2.
61 
62 ### slave_only
63 
64 Switch to 'Master' or 'Slave' PTP mode. Possible values are:
65 
66 - true - PTP slave mode.
67 - false - PTP master mode.
68 
69 Note: Currently, PX-Red supports only slave mode. This flag is future purpose only, when master get supported in PX-Red.
70 
71 The default is true.
72 
73 ### domain_number
74 
75 The domain attribute of the local clock.
76 
77 The domain_number is the primary mechanism for end users and system integrators to isolate the operation of a PTP Instance from PTP messages used in other domains.
78 
79 Range - 0 to 255.
80 
81 The default is 0.
82 
83 ### priority1
84 
85 The attribute priority1 is used in the execution of the best master clock algorithm.
86 
87 Lower values take precedence.
88 
89 Range - The value of priority1 shall be configurable to any value in the range 0 to 255 unless restricted by limits established by the applicable PTP Profile.
90 
91 The default is 128.
92 
93 ### priority2
94 
95 The attribute priority2 is used in the execution of the best master clock algorithm.
96 
97 Lower values take precedence.
98 
99 Range - The value of priority2 shall be configurable to any value in the range 0 to 255 unless restricted by limits established by the applicable PTP Profile.
100 
101 The default is 128.
102 
103 NOTE—In the event that the operation of the best master clock algorithm fails to order the PTP Instances based on the values of priority1, clockClass, clockAccuracy, and scaledOffsetLogVariance, the priority2 attribute allows the creation of up to 256 priorities to be evaluated before the tiebreaker. The tiebreaker is based on the clockIdentity.
104 
105 ### interface
106 
107 String - Network interface on which PTP instance will listen for PTP messages.
108 
109 The default is "eth0".
110 
111 ### PTP Clock
112 
113 String - PTP hardware clock used for Hardware timestamping.
114 
115 The default is "/dev/ptp0".
116 
117 ### status_poll_interval
118 
119 The poll rate at which PTP status get polled from local ptp4l instance.
120 
121 Range - 5 to 50 seconds.
122 
123 The default is 10 seconds.

180 #### steps_removed
181 
182 This represents How many hops the local PTP instance is far from the Grand Master.
183 
184 Suppose, If local PTP instance is directly connected to Grand master (with no Boundry Clock) than value would be '1'.
185 
186 #### parent_port_identity.clock_identity
187 
188 String - MAC based identity of parent PTP clock.
189 
190 In case, if no Boundry Clock (BC) between Slave and GrandMaster, than value of grandmaster_clock_identity and parent_clock_idenity remains identical.
191 
192 #### parent_port_identity.port_number
193 
194 An index identifying a specific PTP Port on a parent PTP Instance.
195 
196 #### grand_master_clock_identity
197 
198 MAC based identity of GrandMaster PTP clock.
199 
200 #### grand_master_clock_quality.clock_class
201 
202 This is the configurable parameter in ptp4l GrandMaster instance.
203 
204 This is the value from Grand Master PTP instance , not from local PTP instance.
205 
206 An attribute defining the TAI traceability, synchronization state, and expected performance of the time or frequency distributed by a Boundary Clock or Ordinary Clock.
207 
208 The clockClass attribute of the GM clock. It denotes the traceability of the
209 time distributed by the grandmaster clock.
210 
211 The default value is 248.
212 
213 Refer Table:4 for clockClass specifications from IEEE 1588-2019 specifications.
214 
215 #### grand_master_clock_quality.clock_accuracy
216 
217 The clockAccuracy is one of the attribute that characterizes a PTP Instance for the purpose of the best master clock algorithm.
218 
219 This is the value from Grand Master PTP instance , not from local PTP instance.
220 
221 It indicates the expected accuracy of a PTP Instance when it is the Grandmaster PTP Instance.
222 
223 The default value is 0xfe.
224 
225 Refer Table 5—clockAccuracy enumeration from IEEE 1588-2019 specifications.
226 
227 #### grand_master_clock_quality.clock_log_variance
228 
229 The offsetScaledLogVariance attribute of the GrandMaster clock. It characterizes the
230 stability of the clock.
231 
232 This is the value from Grand Master PTP instance , not from local instance.
233 
234 The offsetScaledLogVariance is also one of the attribute that characterizes a PTP Instance for the purpose of the best master clock algorithm.
235 
236 The default value is 0xFFFF.
