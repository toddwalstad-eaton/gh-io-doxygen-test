cdsutil Documentation

Utility to provide a command shell that supports session-based activities.
This tool is useful for testing the CDS sessions, particularly to observe activity in other sessions and to see changes in those users or sessions.

This shell session example illustrates the action options for this program, and shows their use in a normal sequence of operations. 

~ $ cdsutil -h
This utility takes no command-line parameters.
Interactive CDS utility. Enter 'help' for help.
Connecting to CDS...
Connected.

list_sessions (ls): list_sessions [token]: List all active sessions. Supply a token from a login session with sufficient privilege, or the one from the latest 'login' will be used.
login (li): login <username> <servicename> <password>: Log into the CDS
logout (lo): logout [session_token]...: Log out a CDS session. If no tokens specified, log out all sessions
session_check (check,chk): session_check [token]: Return permissions granted to that session, none if the session is invalid. If no token provided, return permissions granted to the last token from 'login', or your Linux user ID, if any.
term_sessions (kill): term_sessions [token] [handle]...: Log out sessions, even if not your own. Supply a token from a login session with sufficient privilege, or the one from the latest 'login' will be used. Supply handles returned by 'list_sessions'.
