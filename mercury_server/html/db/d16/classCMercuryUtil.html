Authorize read access to a resource.

There are a few steps to this: 1) Lookup the virtual URI in our map of virtual to real paths 2) Remap the virtual path to a (potentially) real system path 3) Normalize the real path (to screen out any attempts at malicious redirection). The path must exist, but can be either a folder or a file. Will also make sure it does not contain ".." of "/." to prevent access of of relative paths or hidden files/folders 4) Check if the session privileges permit read access of this resource. Normally this is always true if the mapping and normalization passed.

Parameters
virtual_uriThe string value representing the virtual uri (folder or full filepath).
blob_file_accessHolds the BinaryBlob folder configuration information.
session_privilegesThe set of this session's user privileges.
Returns
The real system path to an existing resource.
Exceptions
msg_errWith an exception message if the remapping failed or the resource does not exist.
authorization_errWith an exception message if the read authorization failed somehow.

Authorize write access to a resource.

There are a few steps to this: 1) Remap the virtual path to a (potentially) real system path 2) Normalize the real path (to screen out any attempts at malicious redirection or to affect hidden files). If ok_to_create is True, the boost normalization can't be used (because the resource presumably does not yet exist), so we do a simpler check: We will ensure that the path does not contain ".." or "/.", and throw an authorization_err if this is found. Otherwise, will normalize to handle ".." and other issues. This will also convert any "//" characters to simple "/". 3) Check if the session privileges permit write access of this resource (for C-UD operations). Will throw an authorization_err if this check fails. 4) If ok_to_create is true and the resource does not exist, will create any necessary child folders.

Parameters
virtual_uriThe string value representing the virtual uri (folder or full filepath). If creating a resource, it will be presumed to be a file.
ok_to_createIf true, the resource does not have to exist already and will be created as an empty file or folder. For this to distinguish folders from files, the resource must end with a dot and some file extension after the last path separator if it is a file that must be created.
blob_file_accessHolds the BinaryBlob folder configuration information.
session_privilegesThe set of this session's user privileges.
Returns
The real system path to the existing (or potential) write-authorized resource.
Exceptions
msg_errWith an exception message if the resource does not exist and ok_to_create is false.
authorization_errWith an exception message if the remapping, normalization, or the write authorization failed.
