Detailed Description

Implementation of Mercury "CMercurySubscribeDeviceStatesRequest" message.

The significant part of this message is the DeviceSelector which indicates which Devices should be subscribed to, initially. (This subscription can be updated with the CMercuryUpdateDeviceStatesSubscriptionRequest message, using the CMercuryMessageHandler::subID returned by the CDS when this message is submitted.)

The operation of this Message occurs in the following sequence of steps:

  1. execute(): Subscribes to the CDS; will get a subscription ID when the CDS responds
    • If the DeviceSelector is empty, will respond right away with an error message.
  2. post( _state: SUBSCRIBE )
    1. If there was a CDS error, sets the error code and returns that response.
    2. (Else) Gets the subID from the CDS response
    3. Looks at the DeviceSelector information:
      • Builds a value_state_update subscription request for each Device, submits them to the CDS, and waits for the response to that request (this message _state is now POST).
  3. post( _state: POST )
    1. Looks at the CDS response to the device_state_update subscription requests, and returns an error code (if any), else No Error. (That's all, no information is returned per subscription point.) Also adds the subID to the ProtocolHandler's list and returns the subID in the response.

bool CMercurySubscribeDeviceStatesRequest::post ( response_ptr_t  resp)
virtual

Handle the response to a cds request.

Implements the method inherited from CMercuryMessageHandler.

This method handles the cases of two states

  • SUBSCRIBE: initial response to the subscription ID request from the CDS, of type CManageSubscriberIDs. Will then make individual Channel or Device subscription requests to the CDS
  • POST: response to those subscription requests from the CDS.
Parameters

Returns
True if operation is complete (or error has occurred) and a response has been prepared and sent to the client, False if there is more interaction to occur with the CDS.
