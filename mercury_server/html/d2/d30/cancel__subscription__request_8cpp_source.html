21 /* Test strategy for the "CMercuryCancelSubscriptionRequest" message:
22  *
23  * For all tests, before starting, make sure that the ProtocolHandler is
24  * already holding a known list of subscription IDs in its cache.
25  *
26  * 1) For execute():
27  * This method has 3 cases:
28  * a) For no Subscription IDs provided in the request,
29  * then enter the request (to our mock CDS), and validate that
30  * that request listed all of the IDs in the ProtocolHandler's cache.
31  * Validate that there was no response yet.
32  * b) When there are specific Subscription IDs, including some not
33  * in the ProtocolHandler's cache,
34  * then enter the request (to our mock CDS), and validate that
35  * that request listed only the IDs in both the ProtocolHandler's cache
36  * and the request. Validate no response yet.
37  * c) When none of the specific Subscription IDs are found in the
38  * ProtocolHandler's cache, then validate that nothing is submitted
39  * to the CDS and that the error response is returned.
40  *
41  * 2) post
42  * This method has 3 cases:
43  * a) With no Error, when removing all from the ProtocolHandler's cache,
44  * Validate that all subscriptions are removed from the
45  * ProtocolHandler's cache and that a Success response is returned.
46  * b) With no Error, when removing specific Subscription IDs
47  * from the ProtocolHandler's cache,
48  * Validate that just these subscriptions are removed from the
49  * ProtocolHandler's cache and that a Success response is returned.
50  * c) On Error result, then validate that the error response is returned.
51  */
