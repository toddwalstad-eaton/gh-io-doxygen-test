21 /* Test strategy for the "CMercurySubscribeChannelRealtimeRequest" message:
22  *
23  * 1) For execute(): Since all it does is enter a subscription ID request,
24  * we will do a fairly simple test: ignore the request contents,
25  * make sure it doesn't throw, enter the request (to our mock CDS),
26  * and validate that we get back the right transID (transaction ID)
27  *
28  * 2) post( _state: SUBSCRIBE )
29  * There are some variations on the ChannelSelector (part of the Request) to try,
30  * and an error response from the CDS to test.
31  * a) Empty ChannelSelector: post the valid response (with the subID from the response)
32  * to the ProtocolHandler immediately, without calling the CDS.
33  * Check the Mercury response for no error, right subID, return of True (done).
34  * b) ChannelSelector contains some Device(s): makes a request to the CDS
35  * to subscribe to Channel Updates for those Device(s).
36  * Check the CDS request for right contents, return of False (not done).
37  * c) ChannelSelector contains some Channel(s): makes a request to the CDS
38  * to subscribe to Channel Updates for those Channel(s).
39  * Check the CDS request for right contents, return of False (not done).
40  * d) ChannelSelector contains some Channels AND Device(s): makes a request to the CDS
41  * to subscribe to Channel Updates for those Channel(s) AND Device(s).
42  * Check the CDS request for right contents, return of False (not done).
43  * e) If there was a CDS error, sets the error code and returns that response.
44  * Check the Mercury response for that error, no CDS request, return of True (done).
45  *
46  * 3) post( _state: POST )
47  * Just two cases: with and without error in the response from the CDS.
48  * But first have to take it through one of 2b, c, or d cases to get _state to advance to POST.
49  * a) If Mercury response has No Error and contains the right subID.
50  * b) Else, CDS response contains error, and that should be seen in the Mercury response
51  * In both cases, check the Mercury response for the error code, no further CDS request,
52  * return of True (done).
53  */
