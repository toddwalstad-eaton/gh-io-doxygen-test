Member Function Documentation

void CMercuryUpdateToolkitConfigSubscriptionRequest::execute ( )
virtual

Handle the Mercury request to Update the Subscription entries for one Subscription to OOB updates.

Implements the method inherited from CMercuryMessageHandler.

This method updates the subscribed OOBs for the subscription (ID given in the request) in the CDS using the CSubscribe message. The method processes the request, adding CSubscribeSingle entries, in this order:

  1. OOB subscriptions to be added
  2. OOB subscriptions to be removed

The same CSubscribeSingle structure is used for both additions and removals, with the unsubscribe flag set to differentiate these. After the list of CSubscribeSingle entries is built, it is submitted to the CDS, and we will wait for the CDS response.

There is one error condition that causes an immediate exit, with the error returned to the client: if no subscription ID was provided in the request.

bool CMercuryUpdateToolkitConfigSubscriptionRequest::post ( response_ptr_t  resp)
virtual

Handle the response to a cds request.

Implements the method inherited from CMercuryMessageHandler.

This method handles the cases of three states

  • SUBSCRIBE: response to the update subscription requests from the CDS. If any add_tags were specified, query the CDS for OOB data in this state. Else, respond to the remove_tags with the subID.
  • SUBSCRIBE_POST: response to those Get OOB requests from the CDS of any add_tags. Responds with the complete collection of data and subID
Parameters
respThe response from the CDS to be handled
Returns
True if operation is complete (or error has occurred) and a response has been prepared and sent to the client, False if there is more interaction to occur with the CDS.
