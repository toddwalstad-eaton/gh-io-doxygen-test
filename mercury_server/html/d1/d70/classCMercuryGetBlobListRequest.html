For a given real folder path and pattern to match, get all the files or folders (recursively or not) and add them to the response URI list.

Parameters
real_folderThe base folder in the system to look into
patternThe pattern we will use for filtering.
recurseA flag to indicate that we should recursively fetch the contents of the base folder, and any subfolders.
folders_onlyA flag to indicate that the response list should only contain folder name(s), not filenames, if it is set.
prepend_pathPrepend this virtual path in front of each URI entry we add to the list.
responseThe response to receive the blob list.

Retrieve blob list for the given virtual URI (folder path) and fill it into response.

Parameters
parent_uriThe parent_uri of blob list requested (a virtual folder path).
patternThe pattern filter.
recurseA flag to indicate that, for the response list, the system should recurse through any subfolders of the parent_uri if it is set.
folders_onlyA flag to indicate that the response list should only contain folder name(s), not filenames, if it is set.
no_prefixIf true, insert the full (virtual) URI pathname in each URI of the response Otherwise, insert the virtual prefix and list only the remainder of the file path. The false option is especially useful if calling this function more than once for a response.
responseThe response to receive the blob list.
