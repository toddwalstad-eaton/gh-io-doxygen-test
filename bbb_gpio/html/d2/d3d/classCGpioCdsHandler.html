Detailed Description

BBB GPIO's CDS Handler class (an ASIO Actor, specialized for bbb_gpio).

The communication with the CDS is handled by this class. Many of these methods use synchronous CDS calls (submit_wait()) for the programming flow convenience.

Any asynchronous callback, both solicited and unsolicited, from the CDS will be managed by the handle_response() method. We will use this for subscribing to changes in the GPIO channel(s) in the CDS.

void CGpioCdsHandler::handle_response ( cds::response_ptr_t  resp)

The callback function to handle a solicited or unsolicited response from the CDS.

We will subscribe to changes on our GPIO channel(s), and then expect this callback to be invoked when they are updated; at that time, we will drive the changes to the real GPIO pin(s).

Implementation of the virtual function from px::asio::CCdsHandler

Parameters
respThe generic response message (defined protobuf) from CDS. This is usually then cast to its specific message type.

Enter a subscription for writes to the value of the given Channel, with notification of the calling class.

This function can be called more than once, though the same callback function should be used.

This class will internally manage this subscription.

Parameters
ch_idCDS ID of the Channel to be subscribed to.
channel_value_callbackFunction pointer to the callback that will handle the notifications associated with this subscription. This parameter will be ignored on subsequent calls to this function to add additional channels.
Returns
True on successful creation of the subscription, else False on failure.
