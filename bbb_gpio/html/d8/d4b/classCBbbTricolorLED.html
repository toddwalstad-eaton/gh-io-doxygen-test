Detailed Description

Simple class to implement the TriColor LED of the BBBlack Bacon Cape via the BBB GPIO pins.

Fairly specialized example to illustrate the general principles of working with GPIO pins and their CDS data equivalents.

bool CBbbTricolorLED::initialize_gpio_access ( uid_t  proc_owner)
static

A static function to initialize the file system access for GPIO that the proc_owner will need.

After enabling the GPIO pins for I/O, will set the owner for their file area (eg, /sys/class/gpio/gpio7) to proc_owner for the later I/O actions. This function should be called before px::CUtil::set_gid() changes access rights from root to proc_owner. It is implemented as a static function so it can be called even before this class is instantiated, but still have access to the const assignments for GPIO pins.

Parameters
proc_ownerThe GID of the ultimate process owner, or 0 (root) if none has been assigned.
Returns
True on success, else False if file descriptor open or write failed.

uint32_t CBbbTricolorLED::get_LED_color ( )

Read the I/O pins and establish the "color" the LED is set to.

This will also update the CDS Channel value with the value read.

The color is determined by the 3 GPIO inputs, with Red setting bit 0, Green setting bit 1, and Blue bit 2. So (as the MCL says): 0=Off, 1=Red, 2=Green, 3=Yellow, 4=Blue, 5=Magenta, 6=Cyan, 7=White.

Returns
The value of the color that was determined from the I/O pins.
