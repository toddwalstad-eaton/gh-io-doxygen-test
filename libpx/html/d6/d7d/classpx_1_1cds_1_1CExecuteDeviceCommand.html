Detailed Description

Execute a device command/control operation.

If the command has been registered to that device via px::cds::CSetDeviceCommand, and the relevant device handler has subscribed to px::sub::device_command events on that device, it will get notified.

Client can use px::cds::CGetDeviceCommandResult to find out whether the command is still in progress, succeeded or failed.

At least one of the permissions in the command's px::CDeviceCommand::permissions field as it was registered by its device handler

px::CDeviceCommand px::cds::CExecuteDeviceCommand::command

The command to execute, including any parameter values.

px::CDeviceCommand::device_id must be set to the target device.

The px::CDeviceCommand::handle member is set in the returned message. A nonzero value indicates the command was successfully queued. Likewise, px::CDeviceCommand::status and px::CDeviceCommand::substatus are updated.
