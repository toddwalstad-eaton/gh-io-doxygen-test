Detailed Description

The main I/O manager to host the Power Xpert Toolkit's ASIO framework.

This class manages all server events from one central location, performing asynchronous I/O operations to multiplex data transfers. It handles each of the following (and potentially more in the future) in a central, consistent manner:

  • New connections (FastCGI)
  • New connections (WebSocket)
  • New connections (TCP)
  • Incoming connection data (TCP)
  • Connection errors / closure (TCP)
  • Incoming FastCGI
  • FastCGI errors
  • Incoming serial data
  • Serial port errors
  • Signals
  • Timer expiration

void px::asio::CIoManager::quit ( )

Close all actors and exit the main loop.

This call is invoked (in the common case) in response to an incoming signal to shut down the server. Since the original caller is blocked in run(), it will stop the I/O service and cause run() to return to the caller.

void px::asio::CIoManager::install_temporary_signal_handlers ( )

Register "temporary" signal handlers to catch signals in the startup phase before the run() function is called.

These are in addition to the main signal handlers registered by CIoSighandler. This is optional, but useful if the startup takes a long time to complete, eg because it is discovering devices and channels. If used, these temporary handlers will be removed automatically when run() is called.
