Detailed Description

Log a human user in to the CDS with a username and password.

Client fills in a user name, password, and service name. Currently the service name should always be "mercury".

If successful, the CDS fills in a list of authenticated role/permissions, time/host of last login, number of failed login attempts, and whether the user must change password before logging in again, and sets the token for this session into CRequestResponse::session_token.

If failed, the CRequestResponse status reports an error, and "sessions_limit_exceeded" indicates whether the failure is from exceeding the limit of concurrent logins for that user.

Logins can fail because of nonexistent user, incorrect password, the user has px::CUser::locked_out set, or the user's password is expired and the grace period has passed.

A login session may time out from inactivity (defined as lack of CDS messages using its token), be logged out with px::cds::CLogout, or be force-terminated by an admin with px::cds::CTerminateSessions. The system also terminates sessions when a user's role or password is changed (or the role's permission changes). In these cases the client will receive a px::Cds::CSubscriberNotify with px::sub::subscription_type indicating why the session was closed, whether it has subscribed to anything or not. Login sessions also end when the client that issued the CLogin disconnects from the CDS.
