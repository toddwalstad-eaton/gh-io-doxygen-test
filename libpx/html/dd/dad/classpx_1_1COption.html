Detailed Description

This class represents an individual command-line option: the strings (flag, usage, description text, etc.) associated with it, as well as the actions to take to handle it (initialization, parsing, verification).

parse_func_t px::COption::parse

Parsing function.

This function is called when a command-line option matches the "flag" for this object. The full set of command-line arguments is passed, along with the index of the next argument to be parsed (which can be ignored if the option takes no arguments). The return value is the number of arguments which were "consumed" by this option, i.e. the additional number that should be skipped before we resume parsing. This function may not be set to 0/NULL, as an option without a parse function is useless.

This call throws generic_err if parsing fails for any reason; in that case, usage info will be printed to stderr and the program will exit.

postprocess_func_t px::COption::post_process

Post-processing function.

This function is called after command-line options are parsed. Any option-specific cleanup, verification, etc. that needs to occur prior to running the CDS should be placed here. It may be set to 0/NULL to skip post-processing.

defaultval_func_t px::COption::default_value

Default value "getter" function.

This function is called to get the default value(s) associated with an option. If not 0/NULL, it is invoked when printing out detailed usage information to show the user what the default values associated with each command-line option are. You may return an un-defined string (via boost::optional), which is interpreted to mean "no default value".
