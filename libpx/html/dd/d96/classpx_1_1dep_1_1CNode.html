Detailed Description

A node in the graph.

This can be a source or a target. The only real interface we need to define is a way to index/identify the node. BGL uses a numeric index internally, but it's subject to changes as nodes are added and removed. So to make things simpler for the caller, we utilize an external ID instead, which can be chosen by the caller, and will never need to change.

Dependency type.

This defines what kinds of dependencies are present between this target and one of its sources. We allow for multiple dependency types per-source, since it's possible that a single target might (for example) depend both on a source's real-time value and also on its interval value.

virtual px::dep::CNode::id_t px::dep::CNode::id ( ) const
pure virtual

Get a unique identifier for this node.

The sub-class can implement this in whatever way is convenient. When dealing with channels, for example, it may be convenient to simply re-use the channel's ID here.

The graph code is capable of mapping between its internal structure and these IDs provided by each node. It's up to the caller of the graph functions to map from ID back into the object in question; this is usually easy, since for things like channels that already have an identifier, they're already arranged for easy lookup by ID.

Returns
An identifier for this node which is unique within the context of the graph(s) in which it's used
