Detailed Description

The abstract I/O "Messenger" class that will be implemented in the TCP and UDP connection classes.

An I/O messenger (CIoTransactor or descendant) is a special kind of actor: one that can receive external messages and service them. Not all actors are messengers; for example, signal handlers, timers, or TCP listening sockets don't actually do anything directly in response to external input. But things like established TCP or UDP connections handle input directly, and therefore share this same "messenger" interface.

One of the important features of these Transactors is the use of the outbox message queue; the outbox contains an array of pending message buffers, which are sent one at a time, and in order. After a message has been sent, it is cleared from this queue.
