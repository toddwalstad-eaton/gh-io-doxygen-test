Detailed Description

The Mercury protocol header, which precedes each Mercury message.

This class can be used to validate (parse) or create (serialize) an appropriate Mercury header.

This class conforms to the "Application Protocol Header" (MBAP header) found in section 3 of the "Messaging on TCP/IP Implementation Guide". (True?)

px::asio::iobuf_const_iter_t  end

virtual

Parse and validate that this is a correct Mercury Header.

The validation will perform the following reality checks on the header:

  • Magic byte is 0
  • Version is between min and _max_header_version
  • Header length is correct for its version
  • Header length + Message length == buffer size
  • Message type matches _required_message_type, if that is not set to HG_MSG_NOT_SUPPORT (the default)
  • If requiring a UUID match, will check that; else will read the UUID into the _uuid member here. No checking of the Mercury Message API ID is done.

If the parsing succeeds, then the public class members will contain the data parsed from the header. If the parsing fails, a statement of the reason will be found in _error_message.

If the header fails this validation, a websocket connection should be closed, because the far end sent something inappropriate and may not be valid.

Implementation of the CProtocolObject virtual function.

Parameters
startBuffer pointer to the start of the header.
endPointer to the end of the header + message buffer.
Returns
The result pair, where the first part will be true if the header is valid and false if not; we do not expect to see the indeterminate tribool result after this operation, since we should always receive a complete websocket message before this parse is called. The second part will point to the start of the message on success.

std::vector<unsigned char> px::proto::CMercuryHeader::_uuid

UUID of the target on a Request message, or the source for a Response or published update.

16 bytes in length.

There are two modes for this field when parsing:

  • If set_parsing_requirements() has provided a UUID to match then that value is saved in this field and not overwritten.
  • If not, then the UUID read in when parsing is placed here.

We will apply this UUID when serializing Response or Update output and the _header_version is set to 3.
