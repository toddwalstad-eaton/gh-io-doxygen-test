px::CMultistateMap px::CParameterInfo::multistate_map

Map of string labels to integer/boolean values.

When this field is set, the minimum/maximum should be left empty.

When dealing with an integral or boolean type, we may wish to display values based on some string label rather than the raw numeric/boolean value. For example, a control code may have dozens of possible integral values, each of which should be displayed as a string to the end-user. This map provides the string labels that correspond to the raw values.

boost::optional<px::CValue> px::CParameterInfo::minimum

Minimum value that is valid for this parameter.

When this field is set, the multistate_maps should be left empty.

The UI may choose to use this value as a lower bound to validate the parameter value before sending the command to the device.

If this field is supplied, the type must be the same as the type of the parameter's value itself (i.e. it corresponds to value_type).

boost::optional<px::CValue> px::CParameterInfo::maximum

Maximum value that is valid for this parameter.

When this field is set, the multistate_maps should be left empty. The UI may choose to use this value as an upper bound to validate the parameter value before sending the command to the device.

If this field is supplied, the type must be the same as the type of the parameter's value itself (i.e. it corresponds to value_type).
