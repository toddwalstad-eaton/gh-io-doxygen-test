Detailed Description

UDP Socket Listener class, used to listen on a UDP port.

This class implements a UDP "listener". It listens to a specific UDP port and manages responses to the incoming requests. In the normal case, we receive a request on this port, process it, and send a response (unicast) back to the sender. If the incoming message was broadcast to us, the _want_response flag is cleared and the default response is to send no reply to this message; if you do want to reply to a broadcast input, then you need to set the _want_response flag.

This class does keep a queue of destination endpoints to match the queue of output buffers (messages) to be sent.

This class is not a good choice if you need to send messages out for which you expect a reply (matched somehow to your request), at least as currently designed. However, there is a simple write_push() provided to send a oneway message out.

void px::asio::CUdpListener::write ( std::shared_ptr< io_buf_t >  pbuf)
virtual

Override of the base function to write data to the messenger's output channel's outbox.

This call queues the data to the outbox, which will eventually be sent to the asio channel asynchronously. The reason it is overridden here is to also queue up the current _send_endpoint, which has presumably been filled out by the receiving function with the source's address. This function also checks the _want_response flag, and just drops the packet if not set.

This is the "normal" response function; the write functions in this class let you set a specific target address or broadcast address. (These also end by calling this function.)

Parameters
pbufThe buffer of IO data to be written (ie, the message). Must not be null.

private

Queues the endpoint address to send with each output buffer.

This queue is necessary for UDP messages since there is no single "connection" to the other end, as there is for TCP connections. This queue parallels the _outbox message buffer queue, and must be kept in sync with that queue.

bool px::asio::CUdpListener::_want_response
private

Flag to indicate if we should send a response message.

Will be set true if received was not broadcast or if was received as a broadcast but has been overriden by a call to set_want_response().

If this flag is cleared (ie, received as a broadcast) when we write, we actually just drop any response and send nothing back.
