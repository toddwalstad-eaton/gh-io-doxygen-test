Detailed Description

The Signal Handling actor, which receives and responds to system Signals.

This class implements an I/O actor that notifies the I/O management system when a signal is caught. Practically speaking, this is useful for detecting when the application is being killed, so that a graceful shutdown can be performed.

The signals to be handled are SIGPIPE, SIGHUP, SIGINT, SIGQUIT, and SIGTERM.

This class has been extended to also provide an optional "temporary" signal handler to be used before the ASIO manager is "running". This is a useful option since the startup and device discovery phase (if used) can take quite a while to complete, and this is commonly done before the manager runs and the ASIO framework can catch the main signals registered when this class is instantiated. The temporary signal handler should thus be removed before the manager is run.

void px::asio::CIoSighandler::register_temp_handler ( void  )

Registers an optional "temporary" signal handler, which operates outside of the ASIO framework, and provides signal handling in the early phases of the process startup.

(That is, it is functional before the ASIO manager is "running", and should be unregistered immediately before starting to run the ASIO manager.) Registering these handlers will temporarily replace this class' main handlers.
