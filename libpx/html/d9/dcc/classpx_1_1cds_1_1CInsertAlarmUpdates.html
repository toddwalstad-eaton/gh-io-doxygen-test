Detailed Description

Adds alarm updates in 'requests' directly to the table of alarms, without them needing to be triggered by a change in a channel's value.

No checking is done to ensure e.g. device/channel IDs exist. IDs, indices and global counts are ignored and assigned by the system, unless the 'use_given_ids' flag is true. (Use that for e.g. restoring from backup, or testing.)

CDS fills in 'succeeded_updates' with all of the requested updates that made it into the table, with IDs, indices and global counts filled in if the system assigns them.
