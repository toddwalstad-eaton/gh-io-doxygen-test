Detailed Description

Create one or more devices.

Client fills in a list of Device objects. If "use_given_ids" is false (the default), their "id" members must be unset (use the CDevice default constructor). Their "parent" members must be set. There is always a publisher device (ID 1) so there will always be some parent available. CDS returns the same list of Device objects, with IDs filled in (or fewer, if some could not be created for some reason).

Note
The device objects cannot contain subdevices or channels for this step of device creation; that must be handled in subsequent operations.

px::permissions::install or px::permissions::internal_app

Member Data Documentation

std::vector<px::CDevice> px::cds::CCreateDevice::devices

Device objects containing info on devices to be created.

The "devices" and "channels" fields are ignored. The "id" field must be PX_ID_UNSET (as in the default px::CDevice constructor) unless use_given_ids is true. Successfully-created devices are returned in this list, with "id" and "devices" fields filled out.
