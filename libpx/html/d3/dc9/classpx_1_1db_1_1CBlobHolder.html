Detailed Description

Binary Blob Holder class.

Key/value stores treat the key and value as opaque binary blobs, usually requiring a non-temporary buffer to/from which these objects can be serialized and de-serialized. This simple class is used to hold the results of serialization and de-serialization.

It also bridges the gap between the underlying Toolkit classes (which know nothing about the specific database in use) and the database classes (which know nothing about the Toolkit objects to/from which its data is converted). Currently the database layer is LevelDB, but that could be changed without much trouble, and the classes utilizing the blob holder would be oblivious.

leveldb::Slice px::db::CBlobHolder::get ( ) const

Get a LevelDB slice referring to this blob.

A LevelDB slice is just a thing wrapper to an existing in-memory buffer (pointer + size). As such, it can't be generated directly from a well-typed class; instead, a blob holder is instantiated to store the serialized data for a key or value class, and this call is used to extract a LevelDB slice that refers to it.

While the slice is in use, you must not modify the blob contents.

Returns
A LevelDB slice referring to this blob
