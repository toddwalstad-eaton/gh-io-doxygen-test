Detailed Description

A small Callback handler used for handling responses to submit() and subscription update calls.

Its handling is to post messages from the CDS client's background thread to the application's thread via the ASIO io_service.

Its callback gives the same ASIO spin to subscription updates that it gives to regular responses to submit()'d requests.

It will also redirect subscription updates to the internal callback handler (if used).

Member Function Documentation

void px::asio::CCdsConnectionCallback::cds_response_callback ( std::shared_ptr< px::cds::CRequestResponse >  resp)
virtual

The callback implementation that we need to handle "resp" messages from the CDS.

These messages will take one of 3 forms (as indicated by resp->handle):

  • A response to a submit() request that we made
  • A subscription update for something that either we or the CdsHandler internals subscribed to (keep the internals in mind!).
  • An "unsolicited" message from the CDS itself
Parameters
respThis function will post this "resp" message to the derived CCdsHandler::handle_response() function via the ASIO io_service, so that function will be invoked in the application's thread. If this was a subscription update or unsolicited message, it will also be provided to CCdsHandler::internal_handle_response().
