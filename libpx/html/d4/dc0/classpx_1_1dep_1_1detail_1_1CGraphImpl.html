Detailed Description

Dependence graph (implementation)

This is the actual implementation of the dependence graph; it's buried within its own class for implementation-hiding reasons.

typedef boost::adjacency_list<boost::vecS, boost::vecS, boost::bidirectionalS, param_props, CNode::dep_type_t> px::dep::detail::CGraphImpl::graph_t

Type of our main graph structure.

We use vectors for both the vertex and edge lists for efficiency, because the non-existence of parallel edges is enforced in our own code, and add/remove operations are relative rare. Each vertex also attaches an additional property: our external node ID. Edges track dependency information, such as whether we're dealing with a real- time or interval-based aggregation.

bool px::dep::detail::CGraphImpl::has_cycle ( ) const
private

Detect cycles in the dependence graph.

This utilizes BGL's DFS visitor interface to detect cycles in the graph. It's not quite as simple as doing plain cycle detection, because we have to account for the dependence type on each graph edge: currently, only cycles of real-time dependencies can cause problems, so we discard interval-only graph edges from our search.

returns true if graph contains a cycle
