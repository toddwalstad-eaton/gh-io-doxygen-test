Detailed Description

A simple trigger for CScheduler that fires every interval_minutes, starting at starting_offset_minute after the hour.

For example if starting_offset_minute = 1 and interval_minutes = 5, this will trigger (starting at midnight UTC) at 00:01, 00:06, 00:11, ...

Does guarantee that it will trigger starting at interval_minutes after the offset, but does not guarantee how soon after time; if the application calls in_range() frequently enough, should be 0-59 seconds after the expected time.

bool MinutesTrigger::in_range ( const px::CWallTime &  start,
const px::CWallTime &  end 
)
virtual

Checks whether the "end" time (presumably "now") is within the next triggering window since the "start" time.

As an example, if this trigger was configured for starting_offset_minute = 1 and interval_minutes = 5, given a start time of :12:00 (minutes and seconds; we can ignore day and hour and even UTC vs localtime), if "end" time is :15:55, will return False since this is not yet time to trigger for the next interval. However, an "end" time of :17:33 would return True since it is past :16:00.

Parameters
startThe starting time, presumably in the previous trigger window.
endThe ending time, presumably now, to be checked.
Returns
True if end is at a point where it falls into the "next" trigger interval after the "start" time, or it is precisely on that trigger point.
