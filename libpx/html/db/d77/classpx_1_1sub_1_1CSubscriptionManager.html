Detailed Description

Class for managing CDS subscriptions.

This class encapsulates subscription handling for use by the CDS. It doesn't actually implement subscription-based publication, or know anything about the kinds of things being subscribed or published; rather, it's a lightweight class that can be used to manage a many-to-many mapping of subscriptions between some kind of "subscribers" and some kind of "objects".

In practical terms, the "objects" are typically things like channels and devices, while a "subscriber" is an application-level concept that's exposed through the Toolkit API. This allows a single application to manage any number of separate subscribers on an individual basis, so that (for example) the Mercury server doesn't need to duplicate this functionality in order to support multiple clients that each have their own individual subscriptions.

Performance is important for subscriptions. By far the most common case, and therefore the case that these classes are optimized for, is that of a channel value update which may need to be pushed to any number of subscribers interested in real-time updates. Therefore we don't mind things like the creation and deletion of subscriptions to be somewhat slower, if it helps to optimize the case of subscriber look-up (which could happen thousands of times per second on a busy system).
