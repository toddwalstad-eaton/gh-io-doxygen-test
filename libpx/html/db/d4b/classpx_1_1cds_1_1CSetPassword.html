Detailed Description

Change a user's password.

Client fills in a list of (user ID, password) pairs. The plain-text password may be set, or a hash may be set directly (e.g. one obtained from CGetPassword for backup/restore purposes).

CDS fills in a list of user IDs and result enums.

Changing a password may fail for password-policy-related reasons (e.g. too short); see px::CUserPolicy and px::proto::ChangePasswordStatus.

px::permissions::security_admin. Any logged-in user can set their own password (their existing password must be provided in such a case).

Member Data Documentation

password_type px::cds::CSetPassword::pwtype

Whether passwords are plain text, hashes, or htdigest hashes.

Passwords are stored hashed. If a plain-text password is passed in, it is hashed and then stored. Hashes can be set directly, by changing this member (useful for restoring backups). Hashes are set in the format returned by CGetUserPassword: hex digits for htdigest, printable characters for regular hashes. A user must have px::permissions::security_admin permission to set a hashed password, even their own.
