Detailed Description

Batch of Database Operations.

When modifying a key/value database, this class is used to lump together all of the modifications into a single atomic unit. This is beneficial both semantically (e.g. to ensure that the database isn't partially updated in the case of a crash), as well as for performance reasons. This class maintain the key/value blobs to be live until the clear or delete, because the levelDB write batch require the buffer to be alive.
