Detailed Description

Set a device command/control result after execution.

Usually, the device handler sets the result after the execution. Both the command handle and the device id must be set correctly to find and set the command's result. The queued command entry will be updated with the information provided in this message.

There is no response other than px::cds::RequestResponse::status. Clients can see this result using px::cds::CGetDeviceCommandResult.

At least one of the permissions in the command's px::CDeviceCommand::permissions field as it was registered by its device handler
