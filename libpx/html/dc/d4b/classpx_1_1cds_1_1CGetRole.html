Detailed Description

Get a list of defined roles and/or details about them.

Client fills in a list of role names. If empty, all roles defined in the system are returned. Responder fills in "role_defs", with the role name and the permissions assigned to each role. If any roles in the list don't exist, they are left out of "role_defs". If none exist or the requester lacks permission, an error status is set in CRequestResponse::status.

px::permissions::security_read or px::permissions::security_admin to see all roles. Any other user can use this API but will only see their own role's permissions.
