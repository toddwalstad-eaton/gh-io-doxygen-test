Detailed Description

Erase OOB data items.

Client fills in a list of tag strings. Any OOB data present for such tags is cleared and forgotten. There is no response except overall request status.

varies (usually px::permissions::config_write). To delete an OOB object the caller must have one of the permissions listed in the OOB's delete_or_permission_permission_required member. If that member is empty, px::permissions::config_write is required. If that member contains a px::permissions::unknown entry, nobody is allowed to delete the OOB object.
