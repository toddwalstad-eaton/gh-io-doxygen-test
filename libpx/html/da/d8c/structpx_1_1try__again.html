Detailed Description

There are also a few corner cases in which we'd like to break out of an execution path and restart it, but not due to an "error" per-se.

This is particularly useful in the context of the operations queue, since there may be unusual circumstances where we'd like to insert a new (high-priority) operation into the queue, then back out of the current operation, execute the new one, then re-execute the current operation. This exception is used for those scenarios.
