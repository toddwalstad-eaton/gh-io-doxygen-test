1 certcmd Documentation {#mainpage}
2 ============
3 
4 
5  certcmd is a command line utility to access various certificate management functionalities. This utility communicates with a running certmanagd daemon via libcertmanager interface. <br>
6 
7  The command line usage illustrates the operation options for this program. <br>
8 
9 ---
10 
11  ~ $ certcmd -h
12  -help : Display help info.
13  -list : List the types of services supported.
14  -reload :
15  <--reload network> : Notify the network about change in networks to all services depend on that network.
16  <--reload time> : Notify the time change in time to revoke/add a certificate.
17 
18  certcmd <service id> <target> <action> [<parmeter>]
19  <service id> : Id of the service
20  <target> :
21  - server - action: getcsr/createcsr/applycrt/getkey/revoke/info/detail/details/getcert
22  - ca - action: revoke/add/list/info/details/path/getcert
23  - client - action: revoke/add/list/info/details/path/getcert
24  - config - action: reload
25 
26  <action> :
27  - reload - Only works with 'config' target option but not defined yet.
28  - getcsr - Get the server CSR contents.
29  - createcsr - Create a CSR for a server target.
30  - applycrt <file>
31  - Upload a user given certificate for the server CSR.
32  This will replace the CSR with the certificate given.
33  - getkey - Get the private key of the active server certificate.
34  - revoke [certId]
35  - For server, it revoke the server active certificate. No arg needed.
36  - For ca, it revokes the requested CA certificate.
37  - For client, it revokes the requested Client certificate.
38  - add <path> - Only works for ca/client with arg. it will add the new certificate given.
39 
40  - list - Only works for ca/client with arg. Get the CA/Client certificate list.
41  - info [certId] - For server, it provides active certificate information.
42  For CA/Client, it provides requested certificate information.
43  - detail [certId]- For server, it provides active certificate detailed information.
44  For CA/Client, it provides the requested certificate detailed information.
45  - path - For Server, it returns the active certificate path.
46  For CA/Client, it returns the requested certificate path.
47  - getcert [certId]- For server, it returns the active certificate contents.
48  For CA/Client, it returns the requested certificate contents.
