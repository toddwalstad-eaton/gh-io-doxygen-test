README

This directory holds various files for reference.

Structure of this directory

File name Description
config.daemon This is a sample configuration file for the certmanagd daemon configuration for px-toolkit. Similar file need to be used for its configuration from any adopter.
pki1.conf This is a sample file to define configuration information about a PKI (Public Key Infrastructure) service. This service can act as a master service to provide signed certificates to other services of the device. This service is defined to generate self signed certificate.
https.conf This is a sample file helps to define configuration information about a standard service. In this case this service will get its server certificate signed by the pki1 service.
cnfhook-pki1.hook This is a sample file that describes the PKI1 service server certificate attibutes and configuration options
cnfhook-https.hook This is a sample file that describes the HTTPS service server certificate attibutes and configuration options
pki1_cn.sh This is a sample script which explains how a common name can be generated dynamically for a service
callback-pki1.sh This is a sample file that describes how to handle different system operations, when a operation happens in PKI1 service certificates
callback-https.sh This is a sample file that describes how to handle different system operation, when a operation happens in a HTTPS service certificate
restart_https.sh This is a sample script used to restart all https related px-toolkit system services when a HTTPS certificate gets generated/updated
certmgr-Makefile This is a reference Makefile required to build the certmgr module
