Detailed Description

CPort class.

Despite its naming, the "port" class doesn't interact with the hardware port in any way: that's handled by the I/O subsystem. Instead, the port class wraps together the other things that are logically part of a "port". Most importantly, it's a container for devices. Since the limiting factor in a device handler's operation is communication on a physical port, this grouping by port allows us to make decisions on a per-port basis.

Each CPort-derived object builds a list of protocol-specific (eg, Modbus) CDevice-derived objects from the device_list of the CPort Device instance.

This class is usually derived into a Serial- or Network-specific subclass.

One of the most important duties of the CPort class is to help the IO Messenger keep the serial pipe as full as possible with IO traffic. An important concept here is the transaction_deque_t queues of CIoMessage's; each queue, provided by one of the Devices managed by this Port, consists of one or more CIoMessage entries that the CIoMessenger will send out the Port and handle the response for.

CPort provides the get_transaction_series() method to the CIoMessenger for getting message(s) it will dispatch immediately.

transaction_deque_t * CPort::get_transaction_series ( CPollClock &  tnow)

Get the next transaction series (ie, requests) to be executed for this port.

This is invoked by the I/O messenger when it needs message(s) to send and receive.

What the Port does here is:

  • If no Devices are configured, do nothing
  • If Port-level polling is Disabled, only consider Pass-through here
  • Else (normal case)
    • First ask if there is a pass-through request queued up (as long as we didn't do pass-through last time)
    • If not p-t, then ask each Device what its highest polling priority is
      • Except that we skip the last Device polled, as long as we have more than one Device
      • Ask the winning Device to build and provide the transaction series for that highest priority area.
    • If none has a polling request, give pass-through an opportunity then
Parameters
tnowThe present time, used to recalculate polling priorities.
Returns
Pointer to a valid device or pass-through transaction series queue, or nullptr if no valid device or pass-through has a transaction series to request at this time.
