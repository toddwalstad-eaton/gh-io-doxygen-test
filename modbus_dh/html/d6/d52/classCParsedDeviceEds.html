Detailed Description

Class that parses a JSON-based Device EDS file into the format that the Device Handler needs.

This parsed EDS data is the full set as specified in the EDS file, before any constraints have been applied to remove Channels and other Parameters.

Any data that the Device needs from this EDS must be copied out, as the instances of this class are intended to be destroyed when all interested Devices have finished profiling their device data.

This class uses the Boost JSON-Spirit library for the JSON parsing; see http://www.codeproject.com/Articles/20027/JSON-Spirit-A-C-JSON-Parser-Generator-Implemented Helpful descriptions of JSON format and example are found at http://www.json.org/ and http://www.json.org/example.html

Read in and parse the given EDS file.

The DH should only have to parse this once and retain the parsed data until all interested Devices are done using the data (ie, are done profiling their data).

px::datatype CParsedDeviceEds::parse_eds_multistate_map ( const ::json_spirit::mValue &  m_val,
px::CMultistateMap &  ms_map,
const std::string &  source,
const px::string &  param 
)
private

Parse the text for a Multistate map into the format used by the CDS.

If the value is given in parentheses, it will be used; eg, "Off(0);Normal(1);..." Else, the position starting with 1 will provide the value that is the key for this map. Eg, "first;second;last" will map to 1:first, 2:second, 3:last

Another allowable type is a list of numbers, eg "5;15;30;60", but this could be mixed (eg, "5;15;30;60;Disabled(0)") In this case, the numeric values are used and their text equivalents are the string values. And sometimes the numbers are enumerated with parentheses, eg "10(1);100(2);1000(3);Disabled(15)" in which case the parenthetical values will be used as the keys.

All CValues in the map are (normally signed) integers.

Parameters
m_valThe semicolon-separated list of multistates to be parsed.
[out]ms_mapThe Multistate map loaded with the data after parsing.
sourceThe source ...
paramThe parameter ... There is no return value; if parsing fails, the map will simply be empty or incomplete.
