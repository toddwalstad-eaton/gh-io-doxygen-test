Modbus Serial Application Data Unit.

This structure mimics the protocol definition found in section 2 of the "Modbus over Serial Line Specification and Implementation Guide".

void CModbusSerialAdu::handle ( CDataBlock &  block)
virtual

Handle the data for a request/response, getting it back to the original CDataBlock.

In-line functions.

The typical use case for this call is when dealing with responses; the DH will send a request to a device, then receive a response which needs to be handled (e.g. to update the block's internal data with the received device data bytes).

Many derivatives of this class will not need interaction with a CDataBlock; they should not implement this handler but let the default implementation in this file throw an assertion if they are (unintentionally) invoked.

Parameters
blockA reference to the CDataBlock that will be updated when this function is called.

px::asio::iobuf_const_iter_t  end

private

Parse the received bytes when we don't know the expected size of the response.

Eg, when handling the response for a pass-through packet. In this case, we don't have a defined Modbus Function handler, since we have to be able to handle all possible function codes.

Parameters
startPointer to the beginning of the receive buffer to parse.
endPointer to the end of the receive buffer to parse.
Returns
Tribool value giving the result of parsing. If the CRC check fails, we will assume that we haven't received enough bytes yet and so will return the Indeterminate result.
