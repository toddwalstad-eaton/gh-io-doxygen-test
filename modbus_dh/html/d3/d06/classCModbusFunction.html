Detailed Description

Modbus function code.

This abstraction allows for generic handling of Modbus functions. Each specific function code, e.g. 0x04 ("Read Input Registers") corresponds to one of these function classes.

void CModbusFunction::handle ( CDataBlock &  block)
virtual

Handle the reception of a request/response with this function code.

In-line functions.

This has the same form as CClonableProtocolObject::handle(CDataBlock &) but is a really a different function.

The typical use case for this call is when dealing with responses; the DH will send a Modbus request to a device, then receive a response which needs to be handled (e.g. to update the block's internal data).

Some derivatives of this class will not need interaction with a CDataBlock; they should not implement this handler but let the default implementation in this file throw an assertion if they are (unintentionally) invoked.

Parameters
blockA reference to the CDataBlock (CModbusDataBlock) that will be updated when this function is called.

Reimplemented in CModbusWriteSingleCoilResponse, CModbusReadDiscretesResponse, CModbusWriteMultipleRegistersResponse, CModbusWriteSingleRegisterResponse, CModbusWriteMultipleCoilsResponse, CModbusReadDiagnosticsResponse, and CModbusReadRegistersResponse.
