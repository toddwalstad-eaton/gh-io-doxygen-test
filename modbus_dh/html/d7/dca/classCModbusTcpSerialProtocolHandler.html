Detailed Description

Modbus/TCP Serial Protocol Handler.

This class implements the protocol handler for Modbus/TCP pass-through requests and responses. Incoming Modbus TCP requests are converted to RTU format and passesd down to a serial TTY connection. Mobdus RTU repsonses from the serial connection are converted back to Modbus TCP.

bool CModbusTcpSerialProtocolHandler::parse ( )
virtual

Try to parse the given buffer.

This call parses the input buffer, assigning internal parameters to match the parsed request wherever possible. If parsing is successful, these internal parameters are used to service the request. No further calls are necessary; the protocol handler will take whatever action is required (generating an error, a valid response, etc.) based on the parsing results, and will invoke callbacks on the messenger to drive it to the next state.

Returns
True if parsing is complete (whether it was successful or an error occurred); false if more data is needed before parsing can proceed.

const uint16_t CModbusTcpSerialProtocolHandler::NumberMaxTcpConnections = 3
static

Sets the maximum number of open Modbus/TCP connections for this pass-through port.

Set to 3 by the non-functional requirements, though we have no reasonable fundamental limit.

If this number is exceeded, one of two things happens:

  1. we close the existing connection with the oldest last usage to make way for the new one, or
  2. we reject the connection request

const uint16_t CModbusTcpSerialProtocolHandler::NumberMaxOfServerTransactions = 6
static

Per the Modbus/TCP protocol, this server parameter tells the maximum number of simultaneous requests (transactions) that are allowed on one Connection.

If this number is exceeded, a Server Busy error is returned.

We have no fundamental limit but have chosen a practical one (6); higher than may be advisable but at least this as an upper bound.

For simplicity and practical performance reasons, we will let this upper bound apply to all the Modbus/TCP connections combined into one queue for this port.
