transaction_deque_t * CModbusDevice::next_transaction_series ( )
virtual

Get the queue of "next" Modbus transaction(s) for this Device.

This is called if the Port chooses to give this Device control over the next series of I/O messages. The Device generates and queues one or more I/O message(s) ready for execution and returns a pointer to their queue.

For Modbus, we are usually just queueing up a single transaction to get a Data Block.

Returns
A pointer to this Device's queue of transactions to be sent and received.

uint16_t CModbusDevice::calculate_timeout ( uint16_t  block_specific_timeout)

Utility function to calculate the Rx timeout in milliseconds.

The calculation accounts for the base device timeout, the extension timeout & the block timeout. Also adjusts for ceiling limit.

The base device timeout is from the EDS header. The extension timeout is from the serial port settings. The block timeout is from the EDS block definition (optional).

uint32_t CModbusDevice::calculate_delay ( uint16_t  block_specific_delay)

Utility function to calculate the interframe delay in microseconds.

The calculation accounts for the base interframe delay, the extension delay & the block interframe delay. Also adjusts for ceiling limit.

The base interframe delay is a constant (CModbusDataBlock::MIN_BLOCK_INTERFRAME_DELAY/1000) in msec. The extension interframe delay is from the serial port settings in msec. The block interframe delay is from the EDS block definition (optional) in msec.

interframe delay = base interframe delay value + the extension timeout, unless the EDS block interframe delay is non-zero, then the base interframe delay value is overridden by this value. and interframe delay = block interframe delay value + the extension interframe delay. interframe delay has a 16-bit ceiling limit.

Returns
Calculated interframe delay in microseconds.
