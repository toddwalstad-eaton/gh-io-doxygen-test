Detailed Description

Device Handler's base Protocol Handler class.

This base class extends Libpx's CProtocolHandler for Device Handler protocols, adding class methods necessary for handling messages without being protocol-specific.

virtual void CDeviceHandlerProtocolHandler::handle_send_only ( CProtocolObject *  device_response)
pure virtual

Handle the case where the protocol does not require a response. 

     Some protocols require that slaves do not respond to certain messages.
     These are the "send only" messages. An common example is a broadcast message,
     but there are many others. This method must be called to handle the "send only"
     case - failure to do so will cause the I/O to hang.

virtual bool CDeviceHandlerProtocolHandler::parse ( )
pure virtual

Try to parse the given buffer.

This call parses the input buffer, assigning internal parameters to match the parsed request wherever possible. If parsing is successful, these internal parameters are used to service the request. No further calls are necessary; the protocol handler will take whatever action is required (generating an error, a valid response, etc.) based on the parsing results, and will invoke callbacks on the messenger to drive it to the next state.

Returns
True if parsing is complete (whether it was successful or an error occurred); false if more data is needed before parsing can proceed.
