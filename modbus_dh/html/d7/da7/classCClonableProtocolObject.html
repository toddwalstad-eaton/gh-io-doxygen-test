Detailed Description

Protocol object that's also clone()able.

This allows generic request & response handling, by having a request clone() itself to obtain a suitably-typed response object.

virtual std::unique_ptr<CClonableProtocolObject> CClonableProtocolObject::clone ( )
pure virtual

Create a "clone" of this object.

It doesn't really create a full copy, it just instantiates a new object of the same type as this one. This allows us to have code which (for example) takes a request and reads in a corresponding response, without actually knowing what kind of protocol requests & responses it's dealing with.

Returns
A unique-pointer-wrapped copy of "this" Protocol Object instance.

Implemented in CModbusTcpAdu, CModbusSerialAdu, and CModbusTcpSerialAdu.

void CClonableProtocolObject::handle ( CDataBlock &  block)
virtual

Handle the data for a request/response, getting it back to the original CDataBlock.

In-line functions.

The typical use case for this call is when dealing with responses; the DH will send a request to a device, then receive a response which needs to be handled (e.g. to update the block's internal data with the received device data bytes).

Many derivatives of this class will not need interaction with a CDataBlock; they should not implement this handler but let the default implementation in this file throw an assertion if they are (unintentionally) invoked.

Parameters
blockA reference to the CDataBlock that will be updated when this function is called.
