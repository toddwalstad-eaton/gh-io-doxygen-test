Detailed Description

Internal "parameter" class.

This is a parameter which is needed within the DH internally, but which is not intended to be exposed as a channel to the rest of the Toolkit. One common example is a scaling factor, which is used in equations for "real" channels, but not exposed directly.

Since it is not stored in the CDS, this container has to hold all of the "settings" data necessary for it.

CInternalParam::CInternalParam ( uint64_t  id,
px::CValue &  value,
px::datatype  cds_type = px::datatype::invalid 
)

Constructor to be used when provisionally creating this Internal Parameter from EDS info.

A transient ID will be assigned to this Internal Parameter by the DH. Normally the value will be set to 0 with the Internal Parameter's type. Later, if the Internal Parameter doesn't exist (fails its constraints test), then this Internal Parameter will be removed.

Parameters
idThe unique ID for this parameter.
valueA starting value for the Internal Parameter, of the right type. May typically be 0 or empty string.
cds_typeThe desired px::datatype for this parameter. This value typically comes from the device EDS file.
