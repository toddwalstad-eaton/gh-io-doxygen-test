Detailed Description

Device I/O statistics.

This object contains the logic for handling the I/O result for each message sent to or received from a device. There are two types of "normal" responses:

  1. No errors
  2. ACK response Basically there are 3 types of I/O error results:
  3. Rx timeout
  4. Data validity error (CRC, BCH checksum)
  5. NACK and other non-fatal protocol specific exception response (error code, etc). For each error type two counters are maintained: consecutive and cumulative. When the same error occurs on N consecutive I/O attempts, the state of the communication for the device will be changed to reflect I/O result. Cumulative error counters are incremented to saturation. Cumulative error counters can be reset to zero.

enum CDeviceIoStats::DeviceIoStateEnums

The set of possible Device I/O States we are working with.

Enumerator
IO_STATE_NO_ERROR 

No issues seen with communication with this Device.

IO_STATE_NOT_RESPONDING 

The Device has been determined to be off-line.

IO_STATE_NACK 

The Device has made a series of NACK replies and appears to be functionally unavailable.

IO_STATE_DATA_VALIDITY_ERROR 

The Device has made a series of invalid responses and appears to be functionally unavailable.

IO_STATE_STATE_COUNT 

Placeholder at the end, tells us how many states there are.
