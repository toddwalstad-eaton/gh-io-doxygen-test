Detailed Description

Simple Message class.

This class implements a request/response message, or a request-only message (that does not expect a response).

This Message type is DataBlock-centric, so it is normally used by the device_handler to manage requests for data polling from the supported Data Blocks of a Device.

It may be part of a larger transaction, or series of related messages.

Constructor.

A new simple message is constructed by some handler ('origin'), which donates an already-constructed request object to it.

The response is allocated at construction by invoking request->clone(). (This gives us a response object of the same type as the request object without requiring any additional knowledge of what "kind" of thing we're dealing with).

Parameters
parent_idID of the Device that this message is for.
originThe Data Block (eg, CModbusDataBlock) that is interested in the results of this message. May be an empty pointer for Messages with no Data Block interaction.
requestThe message handler for the request; will be either CModbusSerialAdu or CModbusTcpAdu.
responseThe message handler for the response; will be either CModbusSerialAdu or CModbusTcpAdu.
timeoutmsec until the response to this message should be considered a timeout.
secondary_timeoutafter response begins, msec lapse without additional bytes that should trigger at timeout (0 to disable)
interframe_delayThe additional delay between frames (ie, between this and next request).
