Detailed Description

Modbus Protocol Message class (for raw pass-through message).

This class implements an I/O message for the relatively simple case of a Modbus request/response when we don't know or care about the contents (ie, for a pass-through message).

A new Modbus protocol message is constructed by the response_handler, which then serializes the ready-to-send message bytes into our _wbuf.

Parameters
dev_idThe device that is the target of this message.
response_handlerThe protocol handler that will process the response.
messageThe protocol-based message details.
timeoutThe Rx timeout for this message (in msec). Default is 200 msec.
interframe_delayThe interframe delay for this message (in msec), Default is 20 msec.
