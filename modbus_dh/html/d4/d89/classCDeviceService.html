Find a Serial Port

Queries the CDS for the configuration of a specific serial port below the root device

Parameters
cds_handleHandle to active CDS connection
dev_idRequest a specific port by its device ID if provided here; by default (id == 1) check the publisher for all child ports
Returns
A serial_port_config_t pair of serial port configuration information. The pair contains a px::CDevice object that provides a catalog description of the port device and an associated SerialPortSettings object that indicates serial port specific information such as baud rate.

If no Port are found, the returned pair of shared_ptr objects will be null. ToDo: Similarly, no entry will be made for a Port with no OOB Serial Port Settings.

void CDeviceService::set_pass_through ( const std::shared_ptr< ICdsHandle > &  cds_handle,
const std::shared_ptr< px::CDevice > &  port_device,
const bool  is_pass_through,
const bool  mirror_disabled 
)

Set or clear the pass_through state flag for the specificed port device.

If "mirror_disabled" is true, the disabled state flag on the port device will be set to the same value (true or false) as the pass_through state flag. This option is needed for INCOM ports, where we can't allow pass-through and polling at the same time. For MODBUS ports, polling and pass-through can co-exist.

Note that the CDS will cause changes to the port "disabled" flag to trickle down to the connected devices. The device handler will be notified of device state changes through its reflexive, recursive subscription to device state (via CSerialPort::device_states_callback). There are some cases where a device might have a different "disabled" state than the port, so it is best to let the CDS figure it out and tell us what is going on (as opposed to just assuming that the device state will match the port.)

Parameters
cds_handleHandle to the main CDS Client connection.
port_deviceShared pointer to this port's device data from the CDS.
is_pass_throughSet as True if the port is in pass-through mode, False otherwise
mirror_disabledSet True if disable flag should be set to the same state as pass-through, False to leave it unchanged
