Enumeration Type Documentation

enum svr_log_lvl_t

These values are used to define server "subsystems" whose verbosity can be varied individually, to get more specific information from the server without overloading the user with output.

Note: when updating this array, be sure to update the corresponding array COptions::subsystem_strings.

For Doxygen, each subsystem also has a corresponding "Module" defined in device_handler.hpp and linked to the most important or interesting aspects of the submodule.

Enumerator
SVR_LOG_DEFAULT 

For messages in main and other non-specific areas.

SVR_LOG_LIBPX 

For messages dealing with the interface to libpx.

SVR_LOG_CDS_HANDLE 

For messages dealing with DH interface to the CDS.

SVR_LOG_SETTINGS 

For messages about the Settings or EDSs.

SVR_LOG_CONFIG 

For configuration code area messages.

SVR_LOG_IO_SYSTEM 

For messages about our interface to ASIO and general IO functions.

SVR_LOG_MODBUS_PROTOCOL 

For messages dealing with Modbus handling.

SVR_LOG_SERIAL_IO 

For messages specific to serial IO.

SVR_LOG_TCP_IO 

For messages specific to TCP IO.

SVR_LOG_DEVICES 

For messages dealing with Device discovery and handling.

SVR_LOG_PARAMETERS 

For messages dealing with Channel or Parameter management or updating.

SVR_LOG_SOCKET_IO 

For messages dealing with socket IO.

SVR_LOG_LIST_END

Function Documentation

bool SVR_LEVEL_TEST ( svr_log_lvl_t  subsystem,
int  debug_level 
)

Test if the desired debug level should produce output for the selected debug subsystem.

The result of this test is useful for bracketing debug code that goes beyond the simple print statements like SVR_DEBUG().

Parameters
subsystemThe pertinent entry in the _svr_verb_array[] array. Normally this is defined at the top of your file as SVR_LOG_SUBSYSTEM, but you can deviate from the subsystem fixed there for the rest of the file.
debug_levelThe level (0 to 12 or so) you fix for your debug output at the pertinent part of your code. This will be compared to the level set for the selected subsystem by the command line arguments, eg "-D 5".
Returns
True if debug output is enabled at this debug_level for this subsystem. This is independent of whether or not NDEBUG is defined (for release builds).
