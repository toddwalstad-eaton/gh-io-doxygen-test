modbus_device_handler Documentation

Device Handler for Modbus devices.
Manages device discovery from configuration data, profiles devices, manages polling of data, and handles incoming requests for Pass-Through or configuration changes.

Cut-sheet level Features

  • Connects to and manages data from
    • Up to 32 Modbus/RTU devices on each of 2 ports
  • Uses device EDS files to map device data to MCL Channels for PXI
  • Three operational modes
    • CDS-cached data accessed via the Mercury Server, Modbus/TCP Server, etc
      • (the default and preferred mode)
      • Applies to Modbus/RTU devices
    • Modbus/TCP to RTU pass-through mode
  • PnP Gateway: no additional software required to configure and view attached devices
  • Additional Device functions supported:
    • Control commands
    • Waveform data as COMTRADE files for supported Modbus devices
    • WAGES data (Water, Air, Gas, Electrical, and Steam)
    • User-defined equations to transform device data
