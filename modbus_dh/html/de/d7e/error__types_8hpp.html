enum err_t

Enumerated list of Modbus and other error codes The first part comes from the Modbus specification, which poorly defines several of them.

We use a limited subset.

Text conversion provided by CUtil::get_err_text()

Enumerator
NO_ERROR 
ILLEGAL_FUNCTION 

The function code is unsupported or unknown.

ILLEGAL_DATA_ADDRESS 

The Device reports that the request contains an out-of-range address.

ILLEGAL_DATA_VALUE 

The Device reports that the data was illegal or out of range.

SLAVE_DEVICE_FAILURE 

The Device reports some failure to execute the request.

For Modbus/TCP, this is also known as a "Server Failure" if the server could not execute the request.

ACKNOWLEDGE 

The server (for Modbus/TCP) failed during the execution.

The server accepted the service invocation but the service requires a relatively long time to execute. The server therefore returns only an acknowledgment of the service invocation receipt. The client can next issue a Poll Program Complete message to determine if processing is completed.

SLAVE_DEVICE_BUSY 

The Device or Modbus/TCP Server was unable to accept the MB Request PDU.

This may be because its input queue is already full or it cannot accept requests at this time. The client should retry later.

NEGATIVE_ACKNOWLEDGE 

Not defined in the Modbus spec.

MEMORY_PARITY_ERROR 

A file validity error.

Pertains to File Read and Write codes 20 and 21.

UNKNOWN_MODBUS_EXCEPTION_ERROR 

Not defined in the Modbus spec.

GATEWAY_PATH_UNAVAILABLE 

Invalid or unknown Unit ID (Slave address) in a Modbus/TCP request.

The Modbus/TCP server is the "Gateway".

TARGET_DEVICE_FAILED_TO_RESPOND 

The targeted device failed to respond.

The gateway (Modbus/TCP server) generates this exception. This will be the response if the request times out before the device response can be obtained.
