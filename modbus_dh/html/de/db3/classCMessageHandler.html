Detailed Description

Message Handler ABC.

A "message handler" is capable of creating I/O messages (to be executed by the I/O subsystem), and of taking actions based on the result of message execution.

In more concrete terms: the message handler interface will typically be implemented by a "data block", or some component of a "data block". Each block can generate a message to fetch its own data (for example), and is capable of handling the results of the execution of those messages.

Handle the "completion" of an I/O message (pure virtual here).

When a message generated by this handler is "complete", it will invoke this callback function. "Complete" is in quotes because it doesn't necessarily mean that the message executed successfully; this callback is invoked in cases of I/O errors, timeouts, parse errors, etc. as well as the "happy path" in which 'msg' contains a complete, valid message. 'error' indicates the status of message completion, which the handler can use to determine what action to take.

Implemented in CDataBlock, CModbusDataBlock, and CModbusCommandBlock.
