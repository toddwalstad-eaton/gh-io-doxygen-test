Submit a request for CDS processing.

Sets the transaction_id and callback_handle members. Returns the new transaction_id. If callback_handle is omitted, the CDS will not send any response (and the returned transaction ID will always be 0). If the queue is full, will sleep until there is space if sleep_on_full is true, or discard the request if false. If the request is discarded, transaction ID 0 is returned.

May throw: no_connection (if the connection has not been established or if it drops for some reason), too_much_data (if the request is too large to ever be able to fit in the buffer), generic_err
