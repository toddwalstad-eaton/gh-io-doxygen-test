Constructor, which also initializes the serial port to its defaults. 

If the open() fails, the _port object will be empty, and any
subsequent attempts to use this Serial Port will fail (with asserts).
Note
In Linux, it is possible (especially for root) to open a serial port more than once. While we could "cure" this problem with file locks, that cure is probably more problematic than the problem itself. So just be aware that multiple instances of your process could open the same serial port (ie, share it).
Parameters
managerThe I/O manager object to which this connection is to be bound
port_containerShared pointer to the CPort device container for our px::asio::CIoManager.
pathThe path to the serial device used by this connection
idA unique ID for this serial interface
Exceptions
boost::system::system_errorThe connection could not be initialized
