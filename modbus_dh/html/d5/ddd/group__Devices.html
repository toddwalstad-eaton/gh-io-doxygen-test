Detailed Description

Device States The following are the Device States and Transitions – for the Happy Path:

  • INITIALIZE
    • Initialize from settings, EDS device template
    • Create all potential poll blocks, channels
  • DISCOVER
    • Test against the expected “signature data” and validate that the Device is present, and is the expected type of Device
      • May not be feasible for Modbus
    • Set or Get the persistent Device ID
  • PROFILE
    • Get values once for each potential block
    • Some may fail if not present on that model
    • Device exact model now known
    • Evaluate equation values, EDS conditionals
    • Prune channel/block list as appropriate
    • Push initial channel values
  • RUN
    • Go!

The following are the Failure modes and responses for Device State transitions:

  • May fail to communicate initially
    • Device data and channels just won’t be populated or pushed to the CDS
    • As though the Device doesn’t exist
  • May lose communication after initial discovery
    • Will count consecutive timeouts up to max
    • Then remove the device and all of its data from the CDS
    • PXI and other subscribers will be notified
  • Stay in the DISCOVER state until “found” later
    • Don’t let it degrade Prioritized Polling (much)
