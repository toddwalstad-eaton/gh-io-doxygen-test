Detailed Description

Protocol object base class.

This ABC defines operations which are implemented for each protocol-specific structure, creating a generic interface by which they can all be accessed without regard to internal details.

The generic name – "protocol object" – is because we don't have any real pre-conceived notion of what kinds of "objects" each protocol defines for its tranferred data.

Ideally, you want to define a protocol's data payload as a series of nested protocol objects. To give an example:

  • Modbus-TCP ADU is a protocol object It also contains:
    • Modbus-TCP header, which is a protocol object
    • Modbus PDU, which is a protocol object
  • Modbus-serial ADU is a protocol object It also contains:
    • Modbus PDU, which is a protocol object
    • Modbus-serial CRC, which is a protocol object

Nesting objects in this way makes it easy to develop and test portions of the protocol parsing/serialization code in isolation. It also makes it very easy to re-use portions for different kinds of objects that share some commonality but have slight variances. Top-level code can deal with these objects without caring what, exactly, they are, because they all present a common interface.
