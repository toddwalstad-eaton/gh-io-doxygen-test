Detailed Description

The Device Handler goes through a number of steps to determine its configuration and arrange itself before it enters the run() loop of the ASIO framework.

These steps are:

  • Create a new toolkit::CSysLog object to manage syslogging
  • Register the signal handlers
  • Parse the command-line options
    • Will exit if there are any errors, or if command line usage is requested.
  • Fork off to a daemon process unless prevented by the command-line options.
  • Create the CDeviceHandlerCore (the application's main loop())
  • Create and attempt to open the CDS Handle
    • If the CDS is not yet available, wait and keep retrying
  • Do the necessary configuration for the main loop(), in CDeviceHandlerCore::config():
    • initializes the debug logging subsystem
    • gets the port device information from the CDS and reads their persistent Out-of-Band (OOB) serial port configuration
    • gets the device list for each Port from the CDS, with the device configurations, and adds these to the serial port's device list.
  • Then start CDeviceHandlerCore::main() and
    • Create the Manager for the ASIO framework
    • Create IO Actors for the Signal Handling and Watchdog, and add them to ASIO
    • Create an Actor to manage the CDS and add it to ASIO
    • For each configured Port Device:
      • Create the appropriate Serial Port Actor
      • Add the Port's IO Actor to ASIO and Kick it off
    • Start ASIO running

For more detail on Device startup and EDS mapping to Channels,

See Also
Device handling and Discovery details for the device_handler.

Once entered, the Device Handler stays in CDeviceHandlerCore::main() forever unless we get an exception or are signaled to exit

  • On exit,
    • Stop the main loop() (if still running)
    • Deregister the signal handlers
    • Return the exit code
