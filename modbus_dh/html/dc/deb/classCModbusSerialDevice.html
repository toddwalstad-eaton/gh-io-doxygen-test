)
virtual

Called from the Data Block to handle a device notification.

A slave device can "notify" a network master that the device has some condition that requires the master to send command(s) to the slave. Examples are: events (waveforms, time-stamped trips/alarms), data logs, a setpoints change, device powered-off, Since notification handling is protocol and device-specific, this method is virtual. Refer to the descriptions in the derived classes.

Parameters
commandThe command used to acquire the data block that contains the notification.
block_idThe ID of the data block that is notifying us that it is done.
block_nameThe name of the data block that contains the notification.
dataThe data associated with this I/O response (data block).
errorThe error code associated with the data.
