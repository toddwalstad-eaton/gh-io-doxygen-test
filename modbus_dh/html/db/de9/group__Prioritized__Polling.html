Rationale

  • Original PXG Way: Everything polled equally often
    • Poll everything for each device, in turn
  • But some values are more important than others!
    • Breaker status: want to see changes in seconds
    • Energy: want to see every minute or so
    • Scale factors: hourly/daily?

Features of the Prioritized Polling solution

  • Set priorities for every data value (“parameter”)
    • Aggregated into polling-block priority
  • Pick highest-priority item across all devices, poll it
    • When that response comes in, pick the next highest priority item
    • Repeat forever
  • Priorities float upwards over time while waiting to be polled
  • No set minimum (or maximum) time between successive polls of the same item
  • Each Serial Port kept constantly busy
    • Aiming for 100% utilization

Prioritized Polling procedure at the Port and Device Levels

  • Port level:
    • When ready for the Next Message, (Which is while waiting for current Response)
      • Ask each Device in turn: What is your highest priority level?
  • Device level:
    • When asked for highest priority level,
      • Iterate through all Data Blocks and any Commands,
      • recalculate priority for elapsed time,
      • then answer
  • Port level:
    • Ask the Device which had highest priority level for that Message, and queue it up The following figure shows the sequence for Prioritized Polling:

Simulation of Prioritized Polling

  • A simulation of Prioritized Polling was devised to try out various schemes and ascertain the usefulness
    • See the report and code at px-toolkit/documents/prioritized_polling/
    • The following figure illustrates Prioritized Polling in action for 3 Data Blocks of varying priority:
      Simulation of Prioritized Polling for 3 Data Blocks
  • The figure above illustrates:
    • The dark orange and yellow lines trace two Data Blocks with the same default priority level
    • The blue line traces one lower priority level Data Block
    1. Initially, the orange and yellow lines rise together, until one (the orange) is selected for polling
    2. Then the orange priority is reset to its default. All continue to rise.
    3. The next selected is the Yellow, since it has the highest priority at that time.
    4. Then the yellow is reset to its default level.
    5. The orange and yellow are alternately selected for a while
    6. Then the blue line, which has been more slowly rising the whole time, rises to the point where it is the highest priority Data Block, so it is selected.
    7. Then the blue is reset to its (low) default priority.
    8. This process continues (forever).
