Detailed Description

The Device Handler relies on libpx to provide:

  • The CDS API interface
  • The ASIO framework
  • Definition of px basics:
    • Objects like px::Device and px::Channels
    • Data types like px::string and px::CValue
    • Enum definitions for types, device properties, units, etc
  • The dependence library for determining order of evaluation of Channel data
  • The equation library used in evaluating the Values for many Channels
  • The Mercury protobuf interface, particularly for the Out-of-Band (OOB) data that the Device Handler needs:
    • Serial Port configurations
    • Table of EDSs vs Device Models
    • Device configurations
  • Utility functions:
    • The command line "options" parsing support
    • The debug syslogging functions and macros
