Detailed Description

Implementation of a "command block" for the Modbus protocol.

A Modbus "command block" corresponds to an address range, where the addresses represent either registers (input or holding), discrete inputs, or coils.

This class supports Modbus-serial as well as Modbus-TCP, and allows reads and writes to/from block data using messages with appropriate function codes.

Generate a message which reads/writes this command block from/to the appropriate device.

This call uses _func_gen and _adu_gen to create a Modbus message with suitable request and response objects

Parameters

Shared pointer to the message object that we will pass down into the IO system. The message will not yet be serialized into the write buffer.

void CModbusCommandBlock::handle_io_completion ( CIoMessage &  msg,
err_t  error 
)
virtual

Callback invoked when this DataBlock's messages have been completed.

For the case of no errors, updates the timer and converts and stores base data values. For the cases of errors which are irrecoverable and fatal to the block, bumps the IO error counter and flags this block as invalid if too many consecutive errors are seen. After handling the error cases, invokes the Device callback to inform it that this block is done.
