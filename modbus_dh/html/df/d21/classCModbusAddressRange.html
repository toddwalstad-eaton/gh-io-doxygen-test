Detailed Description

A container class for the range of Modbus register / coil / discrete input which can be addressed by a specific Modbus function code.

The range is dependent on the function code. Since class wraps the Modbus start address, quantity of Modbus items and Modbus function code, it defines the most basic elements of a Modbus message. This class is based on Modbus Application Protocol Specification V1.1b

Besides acting as a container, this class proves useful for: 1) obtaining the maximum quantity of items which can be addressed by a Modbus function in a single Modbus message. 2) validating a Modbus request for function code support and correctness of quantity. 3) performing address matching when there are overlapping ranges of addresses. For example, if parameter X maps to addresses 1000 through 1009, and parameter Y maps to addresses 1010 through 1019, an incoming request addresses 1005 through 1012 will need to fetch both X and Y, even though none of the specific register numbers match between those 3 pairs.
