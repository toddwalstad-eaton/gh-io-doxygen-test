Typedef Documentation

typedef struct eds_header_s eds_header_t

Define the structure to hold the EDS's header data.

This section of the EDS describes the EDS itself and the supported device(s).

typedef struct eds_DBT_entry_s eds_DBT_entry_t

Define the transform for one Data Block Translator row entry from the EDS.

The transform is FROM a Device type (ext_data_type_t) TO the px::datatype for the value. This usually covers more than one mapping within a DBT, but all of the same type.

typedef struct eds_xBlock_entry_s eds_xBlock_entry_t

Define the data that describes one xBlock.

This includes what command and arguments to use in the request, and what Data Block Transform(s) to use to transform from Device to CDS data.

typedef struct eds_cmdinfo_entry_s eds_cmdinfo_entry_t

ToDo: Use px::CDeviceCommand at some point Define a custom CommandInfo struct here because most of the CommandInfo protobuf messages are not defined in px::CDeviceCommand.

typedef struct eds_channel_desc_s eds_channel_desc_t

Defines the structure for Channel attributes from the EDS.

These fields, if provided, override any built-in MCL values for a Channel with this canonical name.

typedef struct eds_param_entry_s eds_param_entry_t

Define the data provided by the EDS for one Channel or Internal Parameter.

The EDS tells us whether this is an Internal Parameter ("Variables") or Channel.
