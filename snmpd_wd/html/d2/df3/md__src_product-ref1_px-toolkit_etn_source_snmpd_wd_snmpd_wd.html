snmpd_wd Watchdog for SNMP daemon

The snmpd_wd is a program to periodically make SNMP requests to a running instance of the snmpd daemon, to validate the responses, and to perform a watchdog reset if it is determined that snmpd has stopped responding correctly (when it ought to be responding).

If snmpd has not been enabled or is not yet fully running, snmpd_wd will not make its SNMP requests.

The reason for a separate watchdog program, rather than just using the normal CDS watchdog approach, is because we don't have full control of the basic snmpd agent (the Net-SNMP codebase) and because we can't be certain which shared MIB extension libraries will be loaded, so can't locate it there.

snmpd_wd Usage

The usage and command line arguments for snmpd_wd are:

``` Usage: snmpd_wd [-D <verbosity>] [-Ns] [-h] [-hv] [-v] [-w <seconds>]

-D : Set the application's debugging verbosity level -Ns : Don't direct log messages to syslogd -h : Print usage information to stderr and exit -hv : Print detailed/verbose usage information to stderr and exit -v : Print version information to stderr and exit -w : Set the CDS watchdog timeout ```

The arguments used in a particular target can be found with the showcfg argument to the /etc/init.d/snmpd script:

``` snmpd_wd -w 90 ```
