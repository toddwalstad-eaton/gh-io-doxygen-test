Detailed Description

Top-level IPC handle.

This class is the long-lived connection handle associated with a single connection between a client and server. One such handle exists on each end of the connection.

It is primarily a wrapper for the underlying command & event queues, but it's useful to contain them in this wrapper class for ease of handling and to prevent file descriptor leakage when errors occur, etc.

Connection factory.

This is a convenient wrapper for the CIPCConnector class, which internally instantiates a connector, uses it to connect and initialize a CIPCHandle object, then destroys the connector and returns the newly-bound handle object.

Parameters
socket_pathPath of the UNIX socket to connect to
cmd_buffer_sizeSize of the shared memory buffer used for commands & responses
evt_buffer_sizeSize of the shared memory buffer used for queueing asynchronous events
Exceptions
generic_errThere was an error synchronizing with the peer process or creating the handle
Returns
A pointer to a dynamically-allocated IPC handle (the caller is responsible for freeing this object)
