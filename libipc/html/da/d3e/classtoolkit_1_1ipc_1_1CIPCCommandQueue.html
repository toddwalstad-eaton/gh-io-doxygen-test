Detailed Description

Command/response queue.

This class implements a shared-memory queue structure for simple bi-directional command and response communication. It uses a shared memory buffer which whose capacity determines the maximum size of a single command or response. Reading and writing takes place directly to/from the buffer, using the exposed shared-memory pointer value. The read and write calls utilize a pair of eventfd() file descriptors to efficiently signal to the other process that the next command/response is ready. Since this is the only system call utilized, and it's an extremely fast IPC mechanism, this queue can be used for very high-performance command/response signaling.
