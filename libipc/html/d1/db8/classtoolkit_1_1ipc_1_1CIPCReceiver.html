Detailed Description

"Receiver" class used to establish new connections

When a CIPCListener object detects a new incoming connection attempt, it creates a receiver object which is bound to that new connection. This transient class can then be used to complete the initialization process and establish a fully-functional handle.

It is treated separately from a CIPCHandle because there are different resources in use by these two classes, and because during the initialization phase (this class) the only useful action that can be taken on the connection is to complete the connection establishment process, which then produces a full-fledged handle.
