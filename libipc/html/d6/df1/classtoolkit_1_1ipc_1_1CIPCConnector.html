Detailed Description

Client IPC connector.

This class is used by client programs to connect to a listener/server. Connection is logically split apart into a couple of stages, but the end result is the establishment of a CIPCHandle object for long-term use. This class is only used transiently, when setting up a new connection.

In libipc, the client is responsible for establishing shared memory buffer sizes for the command and event queues. The values used in init() are inherited by the process at the other end of the connection (the server), and determine the size of the buffers. This is done so that clients can potentially use larger or smaller buffers depending on their maximum expected payload. Buffers should generally be made as small as possible, even though in normal usage RSS will be independent of the maximum buffer size, because the server process has many connections – if all processes map a large amount of memory, we'll run out of virtual address space on 32-bit machines and be unable to handle additional clients.
