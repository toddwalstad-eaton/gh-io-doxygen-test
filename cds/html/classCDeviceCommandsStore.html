Detailed Description

Manage the commands execution lifetime.

We will need to persist the list when we implement the command scheduling. For now, the list only exists in the memory.

bool CDeviceCommandsStore::get_command_status ( const uint32_t  command_handle,
px::CDeviceCommand &  command 
) const

Query the command's status and metadata from our datastore.

This is misnamed, as we get the full command instance back with metadata, not just status.

Parameters
command_handleThe unique ID for this command instance. This was provided when the command was first "executed" (added to this datastore).
[out]commandThe command instance found in our datastore, with metadata and status. If found, this is duplicated from the datastore to be thread safe. If not found, command.status will be set to px::device_command_status::unknown, and, of course, there will be no valid metadata.
Returns
True if found, else False if not.

Sets (usually: updates) the status of this command instance in the queue.

Normally this is provided by the DH after it handles the command and determines the device results.

Parameters
command_handleThe unique ID for this command instance. This was provided when the command was first "executed" (added to this datastore).
statusThe new status for this command instance.
substatusThe new substatus for this command instance.
error_strAn optional error string; expected to be empty on Success.
parametersThe optional output parameter updates. The CDS will check this command's metadata and validate that it has the named parameter(s) and that each is an output parameter before updating it. We only process the name, value, and type fields here; the other metadata is assumed to be unchanged.
