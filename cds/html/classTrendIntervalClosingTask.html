Detailed Description

Task that manages the minute-based closing of the trend interval for all channels of all devices.

This will be called once every 5 minutes (TREND_INTERVAL_MINUTES).

Member Function Documentation

void TrendIntervalClosingTask::execute ( )
virtual

The "execute" function where the real work happens when this task is run.

Run once every 5 minutes, this will:

  • Lower its thread's priority so it doesn't preempt more important tasks
  • Loop through all devices, and
    • For each channel of the device, call its accumulator's close_interval() function
    • If a valid Trend update pointer is returned (because it did just close the interval), store its trended data.
    • Will check the datastore's exit_req flag each time, in case the main loop is nicely asking us to exit.
  • When done, restore the default thread priority
