Detailed Description

The bulk of the work to be done by the CDS is split up into abstracted "operations", which can be generated either directly from a user command (via libpx), or in response to some other operation.

Since the execution of one operation can trigger any number of additional operations, the "operation" forms the basic unit of work for the core CDS engine.

The operations are queued up on main_loop's _oplist, which is a COpList instance.
