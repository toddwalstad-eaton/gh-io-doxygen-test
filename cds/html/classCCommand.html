Detailed Description

Base CDS command class.

Knows how to process the CDS API objects, and subdivide them into operations as necessary. Contains code that is not appropriate for CRequestResponse because it's internal to the CDS.

virtual void CCommand::execute ( COpList &  ops,
CDataStore &  cds 
)
pure virtual

Execute the command specified by the "req" data member.

Execution of a command may occur immediately if the command is relatively simple (e.g. registering for an event type), or it may involve the execution of sub-operations, for example, a CUpdateChannelValue operation can trigger all sorts of different conditions including alarm updates, deadbands, interval trends, etc. For these more complex commands, the sub-operations are added to the operation list, and the caller is expected to execute them all in order to complete the command. The CCommand object must remain valid until the list has been completely executed: operations may internally store handles to this object which could otherwise be invalidated. (see CMainLoop)

Parameters
opsThe current list of operations, which is usually added to by the execution of a command
cdsHandle to the data store
Exceptions
generic_errThrow if any kind of error occurs during execution that can not be worked around or returned to the caller - for example if the wrong number of arguments is provided at the protocol level.
