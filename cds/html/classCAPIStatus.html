Detailed Description

Status.

These are the values returned by the CDS in response to each action. A command may have multiple actions (e.g. CMD_ADD_DEVICE with multiple device names of devices to create), in which case there is a corresponding status for each command.

Objects of this type are intended for use in lieu of the raw enumeration values wherever possible. Despite being first-class objects, they occupy only 1 byte, and impose virtually no overhead.

PUBLIC const char* CAPIStatus::str ( ) const

Getter for this canonical string associated with this status.

Enum to string conversions are supported via an inverse lookup table that maps to the same string entries used by the parse() function. The lookup table is built as a static member, i.e. initialized the first time any part of your program calls this function. After that, string conversions are extremely fast.

A pointer to the string value in an internal lookup table

unsigned int  len

private

Parse an input status string and assign the corresponding status enum.

This function utilizes a gperf-generated perfect hash table to perform an efficient lookup of the given canonical status string, and find its associated enumeration value. Comparisons are case- sensitive, and must match the corresponding canonical string exactly or they will be considered invalid. The result is assigned to _val.

Parameters

lenLength of the string; if 0, length will be determined automatically by looking for a NUL terminator
