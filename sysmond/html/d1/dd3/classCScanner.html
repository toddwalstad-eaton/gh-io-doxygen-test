bool CScanner::launch_scanner_thread ( )

Launches the thread that does the integrity scanner operations.

This will be a run-to-completion task in a new thread, which will allow the system monitor daemon's main thread to continue to run while the scanner checks the file sytem integrity on a slow background thread. Eventually this thread returns the results of the scan, via get_results().

Returns
The result of the launch; will normally be true, unless the thread launch fails, in which case it will be false, and probably something is terribly wrong in the system.

bool CScanner::check_is_done ( )

Utility to check to see if the integrity scanner task (thread) is done.

You should not call get_results() until this returns True, or your main thread will stop and wait on this thread being done.

Returns

CScanner::scanner_result_t CScanner::get_results ( )

Get the results of the integrity scanner thread, once it is done executing.

Use check_is_done() first to ensure that the thread has completed, or this call will make the calling thread wait on this thread's completion.

Returns

void CScanner::request_thread_stop ( )

Utility that lets the calling application request the running integrity scanner thread to stop.

No guarantee of success, and no timeout included here - just best effort. Normally this would only be called on shutdown. The application can then use check_is_done() to see if the thread exited, or wait on get_results() if it is willing to wait.

CScanner::scanner_result_t CScanner::scanner_task ( )
protected

This is the task that does the filesystem integrity scanning.

launch_scanner_thread() will run this task in a separate thread. Logs to the system update log if the integrity checking ends in a failure. Eventually returns the results of the scan.

Returns
Pair containing bools to indicate if the scan completed, and whether or not the file system integrity checking all passed.
