2. Overview

System Monitor

This process serves to monitor the PX Red system for

  • System Integrity
  • Channels of "Metadata" information

This process was formerly known as "integrityd" when it only performed the system integrity monitoring function.

3. Features

  • Integrity monitoring
    • Periodically checks integrity of the software by verifying that the squashfs backing file for the read-only filesystem has not been replaced or tampered with, and that no executables have been put into read-write overlays.
    • Reports status into channel aSoftwareIntegrityCheckFailed on device 1, creating it if it does not exist.
    • Defines command cIntegrityScanNow on device 1; if received, runs an immediate scan instead of waiting for the next scheduled one.
  • Metadata Channel monitoring
