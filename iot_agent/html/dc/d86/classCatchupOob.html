Detailed Description

IoT Agent's Handler for the IotCatchup OOB.

This class manages the reading from, and writing to, the IotCatchup OOB that is used for Trend Catch-up, and holds a copy of the data for the agent's use.

This will be the timestamp which is the beginning of the first span in trend_catchup_spans. This also first checks to make sure that none of the pending catchup intervals are older than what is configured as the maximum catchup window.

void CatchupOob::remove_trend_catchup_interval ( time_t  interval_time,
time_t  cadence_secs 
)

Remove the indicated interval from the trend_catchup_spans, presumably because it has been published and successfully acknowledged.

This ought to be the beginning interval in the first span. If it is the beginning and ending timestamp of a span, that span will be removed. This will write to the IotCatchup OOB. If there are any errors (such as no such timestamp at the start of a span), a warning will be logged.

Parameters
interval_timeThe timestamp of the trend interval to be removed, in seconds since the epoch.
cadence_secsGives the cadence on which the trend intervals will be spaced. Normally this is the realtimes cadence; if it is <= 300s, then we will use 5 minutes as the interval. If > 300s, it must be one of the allowed trend interval values. We will use this to compute the next catch-up interval after removing the given one.

void CatchupOob::purge_old_spans ( time_t  cadence_secs)
protected

Look through the trend_catchup_spans, and see if any entries are older than the allowed catchup interval, and remove any that are too old.

This includes checking within a span and revising the span to fit within the allowed catchup window. If there are any changes, this will write them to the IotCatchup OOB.

Parameters
cadence_secsGives the cadence on which the trend intervals will be spaced. Normally this is the realtimes cadence; if it is <= 300s, then we will use 5 minutes as the interval. We will use this to move the beginning timestamp of a span if the beginning of a span is too old, but the end is not.
