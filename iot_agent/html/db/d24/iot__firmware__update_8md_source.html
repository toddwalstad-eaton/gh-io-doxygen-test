21 ## Product-specific topics, for Device-oriented firmware update
22 
23 The iot_agent supports products that want to manage firmware update for their non-gateway devices.
24 
25 The [IoT OOB Descriptions](../iot_agent_oobs.md) cover all of the OOBs that are necessary,
26 and that are mentioned in this section.
27 
28 ### Product-specific Image Update App(s)
29 The critical piece of product-specific Firmware Update is that the product must supply
30 one or more Image Update apps to handle the
31 three "actions" for firmware updating: "make_version_oob", "check_version", and "update", using
32 the API described in [update_linux_image.h](../update_linux_image/include/update_linux_image.h)
33 for all Image Update apps (of which the program that updates the gateway's LinuxImage is
34 just one example).
35 
36 ### Product-specific ImageVersion OOBs
37 
38 The product-specific Firmware Update app must generate the ImageVersion/product/type/version OOB(s)
39 for the supported devices.
40 
41 The way a product tells the iot_agent which device models have Firmware Update support, and which
42 app to use for the model, is by one or more mageUpdateApp/product OOB(s). These OOB(s) provide
43 a simple mapping from canonical device model to the specific Image Update app name.
44 
45 ### Using a gateway-based Image Update app for multiple devices
46 
47 Sometimes it is convenient to address more than one device for Firmware Updates, especially
48 when the number of such devices attached to a gateway is so big that updating them individually
49 from PX White would be objectionable. Another example of this is when the gateway is capable of
50 doing parallel updates of its devices.
51 
52 The solution to this is to direct the Firmware Update command from PX White to gateway itself,
53 but with an image type different than LinuxImage. To make this work, the product must add
54 an ImageVersion OOB that maps the gateway device, device Image Type, and device Image Update
55 app name together, so the iot_agent can address the correct Image Update app when it gets
56 a Firmware Update command for the device Image Type. See the example
57 [ImageVersion/PX-Red-KVM/Parallel_Image/AnyVersion](https://bitbucket-prod.tcc.etn.com/projects/LNXTK/repos/product-test/browse/rootfs/etc/cds/platform/82_parallel_update_image.json)
58 in the product-test code; the most important parts to specify are "image_name" and "image_update_app" fields.
59 
60 There is some test code to exercise this in a product_test build in [test__fw_update.cpp](../source/test/test__fw_update.cpp).
