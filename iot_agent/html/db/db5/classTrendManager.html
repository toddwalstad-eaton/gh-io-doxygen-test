Detailed Description

Class to manage the Trend publishing and other trend-related services.

Has "friend" access to CIotDeviceClient's protected data.

enum TrendManager::catchup_results

Set of possible results for the trend catchup thread.

Enumerator
NOT_RUNNING 

The default, not-running state of the catchup thread.

INITIALIZED 

The catchup thread was successfully launched.

DONE_SUCCESS 

The success result; all catchup has been completed.

INTERRUPTED 

Had more catchup results to get, but thread was asked to exit.

DONE_ERROR 

Thread has stopped, or failed to launch, with an error.

bool TrendManager::publish_channel_trend_values ( uint32_t  cadence_secs,
time_t  requested_interval_time = 0 
)

Build the IOT_DATA_TREND_ITEM messages for all devices, using the latest, or requested time, trend values from all trended channels, and send it to the Azure IoT Hub.

Sends one message per device, then ensures that the batched messages are all sent now.

Parameters
cadence_secsGives the cadence on which the trend interval will be based. Normally this is the realtimes cadence; if it is <= 300s, then we will use 5 minutes as the interval. If > 300s, it must be one of the allowed trend interval values.
requested_interval_timeIf at the 0 default, use the latest trend data, otherwise, request data for that time.
Returns
True if successfully built and sent the message for any devices with trend data, else False (eg, if there is no recent trend data or encountered an error).

void TrendManager::request_thread_stop ( )

Utility that lets the calling application request the running trend catchup thread to stop.

No guarantee of success, and no timeout included here - just best effort. Normally this would only be called on shutdown. The application can then use check_is_done() to see if the thread exited, or wait on get_results() if it is willing to wait.
