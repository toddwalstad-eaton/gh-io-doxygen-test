Build Flag and Configuration Items

There are multiple build flags and configuration items for IoT Agent depending on features it provides:

How to enable the IoT Agent in your product

Follow step 1 below to enable building and running the IoT Agent application.

Step 1: Enable building the IoT Agent

Edit your project root config file and add the following entries. Rebuild your project. 

###############################
## Eaton Software
###############################

# Enable the IoT agent
CONFIG_EATON_IOT_AGENT?=y
export CONFIG_EATON_IOT_AGENT

This will build the project with default IoT Agent Configurations. Default values of configurations are mentioned below along with description of each config flag.

How to customise the configurations for IoT Agent in your product

To enable/disable different functionalities of IoT Agent, enable/disable the config flags and update the variables as per the product requirements.

Select IoT Agent Transport

Select exactly one of the below transports for your product. Default in PXRed is AMQP WEBSOCKETS.

  • MQTT over Websockets ``` CONFIG_EATON_IOT_MQTT_WS?=y export CONFIG_EATON_IOT_MQTT_WS ```
  • AMQP over Websockets ``` CONFIG_EATON_IOT_AMQP_WS?=y export CONFIG_EATON_IOT_AMQP_WS ```

Enable message statistics

Enable Iot Stats to get the statistics for messages sent to Azure ``` CONFIG_ENABLE_IOT_STATS?=y export CONFIG_ENABLE_IOT_STATS ```

Set the cadence for realtime data publish

Config variable to set the IoT cadence, defaults to 5 secs. Allowed values are < 300s (eg, 1, 5, 15, 30, 60s) and (in seconds) 5min, 15min, 30min, 60min, 2h, 4h, 8h, 12h, or 24h ``` CONFIG_IOT_REALTIME_CADENCE_SECS?=5 export CONFIG_IOT_REALTIME_CADENCE_SECS ```

Set the trend catchup hours

Build variable to set the maximum number of hours of trend data that will be published to catch up after a disconnect or outage. 0 means no catch-up. Default is 72 hours = 3 days. ``` CONFIG_IOT_MAX_TREND_CATCHUP_HOURS?=72 export CONFIG_IOT_MAX_TREND_CATCHUP_HOURS ```

Enable Blob upload to Azure

Build variable to enable the file upload as a blob to azure blob storage using the IoT Agent ``` CONFIG_IOT_ENABLE_BLOB_UPLOAD?=y export CONFIG_IOT_ENABLE_BLOB_UPLOAD ```

When blob upload is enabled, the primary file path, max file size and a timeout duration needs to be set. ``` ifeq (n,y)

Set the initial path where files to upload to blob storage are stored

Default in PXRed is the /flash/ directory

CONFIG_IOT_BLOB_UPLOAD_PATH?=\/initlo\/data\/ export CONFIG_IOT_BLOB_UPLOAD_PATH

Put a limit on the max file size allowed to be uploaded.

This is for the cybersecurity purpose and to prevent IoT Agent

from consuming huge memory if the file is too large

Default limit by PXRed is 20MB

CONFIG_IOT_MAX_FILE_SIZE_BYTES?=20971520 export CONFIG_IOT_MAX_FILE_SIZE_BYTES

Timeout the blob upload process if it takes too long

0 means no timeout set, defaults to 600s i.e. 10 mins

CONFIG_IOT_BLOB_UPLOAD_TIMEOUT_SECS?=600 export CONFIG_IOT_BLOB_UPLOAD_TIMEOUT_SECS endif ```

Enable publishing realtime data as trend actuals

Have all realtime data published as trend actuals every time, at the trends cadence ``` CONFIG_RT_AS_TREND_ACTUALS?=y export CONFIG_RT_AS_TREND_ACTUALS ```

Set the SAS token lifetime

Sets the interval at which the SAS token will need to be refreshed. The actual refresh will happen at 80% of the interval defined here. If not defined, the default is 1 hour (3600 seconds) and the actual refresh is in 48 minutes Defaults to 304 secs for MQTT and 273 secs for AMQP(to get 243 effectively) ``` CONFIG_IOT_SAS_TOKEN_REFRESH_SECS=273 export CONFIG_IOT_SAS_TOKEN_REFRESH_SECS=273 ```
