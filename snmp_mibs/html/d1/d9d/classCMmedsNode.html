Detailed Description

Class to hold the information for one node (scalar or row column) created from .mmeds data, which allows an SNMP request to be fulfilled, normally by getting the associated CDS Channel Value.

typedef std::function<bool ( CMmedsNode &, const px::proto::MibObject &mmeds )> CMmedsNode::validate_special_node_t

Define the functor to validate a "special" node, when this is its Translation Type.

Usually fulfilled by its table entry class (derived from CMmedsNodeTableEntry), which already has access to the CDS Handler, device ID, and already added nodes for this entry.

param node_data Reference to "this" node param mmeds The mapping info for this one node from the .mmeds file (a protobuf structure). return True on success (were able to map to valid CDS Channel or constant data), If False, then this node was not successfully mapped and should not be retained.

Factory function to create a CMmedsNode given the Device and MIB Mapping EDS data for one MIB scalar or table column object.

Parameters
cds_handlerPointer to the CDS Handler, used to lookup and get Channel data.
px_deviceThe CDS Device data for a valid device; we will check its list of Channels.
mmedsThe mapping info for this one node from the .mmeds file (a protobuf structure).
validate_special_node_funcOptional functor to determine if "special" nodes are valid; will be invoked if the Translation Type is "special".
row_indexThe row/secondary index of the MCL name to be checked. Default 255 means not supplied.
Returns
Shared pointer holding the CMmedsNode, if it was created successfully. However, it frequently happens that no successful mapping from EDS to Device Channel data can be made, and so the returned pointer must be checked to make sure that it isn't empty (empty meaning that no such node can be created).

Try to map from the MIB Mapping EDS data for this scalar or column to CDS or constant data.

If successful, the members of this object will hold the information necessary to later fulfill SNMP GET or SET requests.

Parameters
cds_handlerPointer to the CDS Handler, used to lookup and get Channel data.
px_deviceThe CDS Device data for a valid device; we will check its list of Channels.
mmedsThe mapping info for this one node from the .mmeds file (a protobuf structure).
validate_special_node_funcFunctor to determine if "special" nodes are valid; will be invoked if the Translation Type is "special".
row_indexThe row/secondary index of the MCL name to be checked.
Returns
True on success (were able to map to valid CDS Channel or constant data), If False, then this node was not successfully mapped and should not be retained.

void CMmedsNode::set_units_scalar ( float  scale_factor)

An optional setter for _units_scalar in MIB nodes that need scaling because of non-unity SI units, such as kilowatthours (x1000.0) or centiseconds (x0.01).

If scale_factor is 0 or 1.0, _units_scalar will be unset.

Parameters
scale_factorThe scalar that is to be applied to numeric values in fill_mibvar_value()
