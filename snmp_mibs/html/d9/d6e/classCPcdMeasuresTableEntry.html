Detailed Description

Container for data for one row of the pcdMeasuresTable, providing the available measures (column) information for the Device described by this row.

May be instantiated when a Device is added to our table.

bool CPcdMeasuresTableEntry::check_add_table_entry ( std::shared_ptr< CPcdMib >  pcd_mib_instance,
const px::CDevice &  px_device,

)
static

Factory function which checks to see if we need to add a pcdMeasuresTable entry for the given Device.

If we already have an entry for this Device, does nothing. Else will process the possible pcdMeasuresTable columns ("nodes") for this Device to see if we have any valid columns. If yes, will add this entry to the table.

Parameters
pcd_mib_instancePointer to the overall PCD MIB class, which holds valuable items like the CDS Handler, MIB Mapping EDS data, and this MIB Table.
px_deviceThe CDS Device data for a valid device; we will check its list of Channels.
settingsThe set of Channel Settings for this Device; we need it to validate some of the columns of this table.
Returns
True if an entry was added for this Device; False if it already existed, or we failed to get any valid columns for it.

bool CPcdMeasuresTableEntry::validate_special_nodes ( CMmedsNode &  node,
const px::proto::MibObject &  mmeds 
)

Table-specific function to validate a "special" node, when this is its Translation Type.

Given as a functor of CMmedsNode::validate_special_node_t type during node validation, that code will call back to here so we can determine if this node is valid or not, using, among other things, the nodes already added to this row entry.

For the pcdMeasuresTable, the "special" nodes are the Voltage text labels, using the :units of the found Channel else constant text.

Parameters
node_dataReference to the node we are validating; some of its fields will be updated if this is determined to be a valid node.
mmedsThe mapping info for this one node from the .mmeds file (a protobuf structure).
Returns
True on success (were able to map to valid CDS Channel or constant data), If False, then this node was not successfully mapped and should not be retained.
