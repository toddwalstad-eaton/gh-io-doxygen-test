Detailed Description

Container for data for one row of the pcdSensorTable, providing one "sensor" data point as described by this row.

A "sensor" is a channel which measures some non-electrical quality (eg, temperature or humidity). Will be instantiated when a Device with a Channel matching the EDS description for this table is processed.

Factory function which checks to see if we need to add any pcdSensorTable entries for the given Device.

Will loop through all of the Device's Channels to see which, if any, match the pcdSensor criteria in the EDS, and create a row for any matching Channels. If we already have an existing entry for this Device + Sensor Channel, does nothing. Else will process the pcdSensorTable columns ("nodes") for this Device and add the columns; for this table, every row entry has every column.

Parameters
pcd_mib_instancePointer to the overall PCD MIB class, which holds valuable items like the CDS Handler, MIB Mapping EDS data, and this MIB Table.
px_deviceThe CDS Device data for a valid device; we will check its list of Channels.
settingsThe set of Channel Settings for this Device; we need it to validate some of the columns of this table.
Returns
True if an entry was added for this Device; False if it already existed, or we failed to get any valid columns for it.

bool CPcdSensorTableEntry::validate_special_nodes ( CMmedsNode &  node,
const px::proto::MibObject &  mmeds 
)

Table-specific function to validate a "special" node, when this is its Translation Type.

Given as a functor of CMmedsNode::validate_special_node_t type during node validation, that code will call back to here so we can declare that the 3 column nodes are always valid.

For the pcdSensorTable, the "special" nodes are the 3 columns of accessible data, from the Channel name, value, and units. All 3 are always present and "valid".

Parameters
node_dataReference to the node we are validating; some of its fields will be updated if this is determined to be a valid node.
mmedsThe mapping info for this one node from the .mmeds file (a protobuf structure).
Returns
True on success (were able to map to valid CDS Channel or constant data), If False, then this node was not successfully mapped and should not be retained.
