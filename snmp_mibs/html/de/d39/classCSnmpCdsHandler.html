void CSnmpCdsHandler::channel_settings_callback ( const px::asio::CCdsDevice &  container,
const std::vector< px::CChannelSettings > &  settings 
)

Application callback invoked by CdsHandler when it gets the ChannelSettings for a device.

The CdsHandler doesn't store the ChannelSettings, so we use this to store a map per device of channel names to their IDs (so the MIBs can look up the channels easily). Will be give to CdsHandler and invoked only if want_channels is set to True.

This function will call each interested MIB and pass the settings along with the corresponding device's container, so the MIB can do what it wants with the settings data.

This only needs to be called once, but it is possible that it will be called more than once for (possibly unrelated) device changes. Function template: cds_ch_settings_callback_t& channel_settings_callback

Parameters
containerA read-only reference to the container of CDS data for that Device.
settingsThe array of ChannelSettings from the CDS; should be a complete set for this Device.

std::vector< px::CChannelSettings > CSnmpCdsHandler::find_matching_channels ( const std::vector< px::CChannelSettings > &  settings,
const std::string &  mcl_names 
)
static

For the given set of Channel settings, determine all (if any) of the given MCL Channel name(s) that match the name of a Channel in the settings.

The MCL Channel name(s) may contain the '*' wild-card character; matches will be made for any Channel name up to that wildcard.

Parameters
settingsThe array of Channel Settings to be checked (from one Device).
mcl_namesOne or more MCL Channel name(s) in a semi-separated list. If a name ends with '*' then it is a wildcard and we will match any Channel name up to that character.
Returns
An array of matching Channel Settings; if no matches, then returns an empty array.
