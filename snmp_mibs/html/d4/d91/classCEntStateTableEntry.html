Detailed Description

Container for data for one row of the entStateTable, providing the status information for the Device described by this row.

Will be instantiated when a Device is added to our table.

char CEntStateTableEntry::entStateLastChanged[32]

A character string containing the timestamp of last change.

This is the date and time when the value of any of entStateAdmin, entStateOper, entStateUsage, entStateAlarm, or entStateStandby changed for this entity (device / row entry). Has the DateAndTime form: "2d-1d-1d,1d:1d:1d.1d,1a1d:1d" For example, Tuesday May 26, 1992 at 1:30:15 PM EDT would be displayed as: "1992-5-26,13:30:15.0,-4:0"

long CEntStateTableEntry::entStateUsage

The usage state for this entity (device / row entry).

This object refers to an entity's ability to service more physical entities in a containment hierarchy.

A value of 'busy' means that the entity is unable to handle any additional entities being contained in it.

Enums: unknown(1), idle(2), active(3), busy(4)

Entities that are unable to ever service any entities within a containment hierarchy will always have a usage state of 'busy'.

This latter is our case: always ENTITYUSAGESTATE_BUSY

Definition at line 139 of file ent_state_table_entry.hpp.

uint8_t CEntStateTableEntry::entStateAlarm

The alarm status for this entity (device / row entry).

A value of 'unknown' means that this entity is unable to report alarm state.

If no bits are set, then this entity supports reporting of alarms, but there are currently no active alarms against this entity.

Bit flags: unknown(0), underRepair(1), critical(2), major(3), minor(4), warning(5), indeterminate(6)

Only needs 1 byte for storage for this set of bit flags (size ENTITYALARMSTATUS_BYTES), so using uint8_t to avoid endianess issues
