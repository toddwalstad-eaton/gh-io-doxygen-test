Detailed Description

OPC UA server core class.

This class is used to handle overall program set-up/shut-down, initialize semi-global objects, perform the main processing loop and handle callbacks.

OPC UA onRead callback.

Registered by the COpen62541Wrapper::register_value_node_callbacks and called by open62541 stack when OPC UA client requests reading from the server.

This method is defined static, to be common to all server instances. This is used to overcome the limitation of the open62541 stack. The method works like a dispatch method to invoke private callbacks of all registered server instances.

void opcua::server::COPCUaServer::execute ( )
virtual

Execute the server core's main loop.

This performs the real work of the server. This call will block indefinitely, and only returns when the server has encountered some sort of non-recoverable error, or has been shut down by calling the stop method from external thread or signal handler.

Make sure that you have called browse methods before trying to execute the server.

Exceptions
opcua::server::COpen62541Wrapper::execution_errif the OPC UA server failed its execution.

void opcua::server::COPCUaServer::stop ( )
virtual

Stop the server main loop.

This sends the OPC UA server a signal to interrupt the main infinite loop.

This method is supposed to be called from within POSIX signal handlers of SIGKILL and/or SIGTERM signals. Or from another thread requiring the server termination.

OPC UA onRead callback.

Member callback implementation. Called from on_read_cb as part of the workaround solution for limitations of C open62541 implementation.

The callback gets the UA Node ID of the request, find CDS channel ID mapped to this UA Node ID and fetches value from CDS.
