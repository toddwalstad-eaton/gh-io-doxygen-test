opcua_server PX-Red OPC UA Server

Data server for OPC UA framework

Features

To see the feature set implemented in current version, visit the GTW 0.1-Peda specification Confluence page.

Under the hood, the server is using open62541 stack, an open source C implementation of OPC UA, wrapped in a wrapping layer, implemented by #ref COpen62541Wrapper class.

Class usage

In order to successfully start the OPC UA server capable of propagating values between CDS and OPC UA clients, you need to follow these steps:

  1. Create instalce of the class (multiple server instances are allowed).
  2. In your application use the stop method in any event handlers, that should cause termination of the server.
  3. Call the create_map method to create the UA Node structure, mapped to CDS device tree.
  4. Call execute method to start the server main loop. This method will not exit, until the server is stopped.

Known limitations

Right now, after shutting down the server, it is not possible to restart it again, without destroying the whole object.
