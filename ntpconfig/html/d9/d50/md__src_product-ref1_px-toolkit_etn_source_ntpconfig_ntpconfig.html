`ntpconfig` program

The standard NTPD daemon does not save command-line configuration options to the NTP configuration file (ntp.conf). The Eaton ntpconfig NTP configuration utility saves a select list of command-line options to the NTP configuration file.

NTP Config Utility - Command-Line Rules for Saving Options to the NTP Config File

  1. Invoking any of the defined Save Command-Line Options will update the configuration file.
  2. Option parameters MUST be bracketed by double quotes (").

    OK: ntpconfig -restrict "128.4.0.0 mask 255.255.0.0 #Allow DCnet clients"

    NOT OK: ntpconfig -restrict 128.4.0.0 mask 255.255.0.0 #Allow DCnet clients

    Note that the '#Allow DCnet clients' comment text will also be added to the configuration file.

  3. An option parameter may be invoked multiple times.

    For example, to add a pool of servers to the ntp.conf file, the following command-line format could be used:

    ntpconfig -S "0.us.pool.ntp.org" -S "1.us.pool.ntp.org" -S "2.us.pool.ntp.org"

  4. Invoking an option will delete all instances of that option from the current configuration file.

    For example, if the current ntp.conf file has the following "restrict" options:

    restrict default noserve restrict 127.0.0.1 restrict 128.4.0.0 mask 255.255.0.0

    Then after invoking the following "restrict" command-line option,

    ntpconfig -restrict "128.175.0.0"

    The new ntp.conf file would contain only one "restrict" option:

    restrict 128.175.0.0

  5. Invoking an option with no parameters will delete all instances of that option from the current configuration file.

    For example, if the current ntp.conf file has the following "restrict" options:

    restrict default noserve restrict 127.0.0.1 restrict 128.4.0.0 mask 255.255.0.0

    Then after invoking the following "restrict" command-line option,

    ntpconfig -restrict

    The new ntp.conf file will have no "restrict" options.

Option Name Required Command-line Format

  1. config file Y -c "filename" 
                  example: -c "/etc/ntp.conf"
  2. automax -am "[logsec]"
  3. broadcast -br "address [autokey] [burst] [iburst] [key 1-65534] [minpoll 4-17(6)] [maxpoll 4-17(10)] [noselect] [preempt] [prefer] [true] [ttl 1-254(127)] [version 1-4(4)]"
  4. broadcastclient -bc "[novolley]"
  5. broadcastdelay -bd "seconds"
  6. delete servers -DS "dummy"
  7. crypto -crypto "[cert {file}] [leap {file}] [randfile {file}] [host {file}] [sign {file}] [ident {scheme}] [iffpar {file}] [gqpar {file}] [mvpar {file}] [pw {password}]"
  8. driftfile -drift "driftfile [ minutes [ tolerance ] ]"
  9. filegen -filegen "name [file {filename}] [type {typename}] [link | nolink] [enable | disable]"
  10. fudge -fudge "127.127.t.u [time1 {sec}] [time2 {sec}] [stratum {int}] [refid {string}] [mode {int}] [flag1 0|1] [flag2 0|1] [flag3 0|1] [flag4 0|1]"
  11. keys -keys "keyfile"
  12. manycastclient -manycc "address [autokey] [burst] [iburst] [key 1-65534] [minpoll 4-17(6)] [maxpoll 4-17(10)] [noselect] [preempt] [prefer] [true] [ttl 1-254(127)] [version 1-4(4)]"
  13. manycastserver -manycs "address [...]"
  14. multicastclient -multicc "address [...]"
  15. peer -peer "address [autokey] [burst] [iburst] [key 1-65534] [minpoll 4-17(6)] [maxpoll 4-17(10)] [noselect] [preempt] [prefer] [true] [ttl 1-254(127)] [version 1-4(4)]"
  16. restrict -restrict "address [mask {mask}] [flag][...]"
  17. server -server "address [autokey] [burst] [iburst] [key 1-65534] [minpoll 4-17(6)] [maxpoll 4-17(10)] [noselect] [preempt] [prefer] [true] [ttl 1-254(127)] [version 1-4(4)]" 
                   example: -server "10.222.50.245 prefer minpoll 4 maxpoll 6"
  18. statistics -stats "name [...]"
  19. statsdir -statsdir "directory_path"
  20. tinker -tinker "[allan {allan}] | dispersion {dispersion} | freq {freq} | huffpuff {huffpuff} | panic {panic} | step {step} | stepout {stepout} ]"
  21. tos -tos "[ ceiling {ceiling} | cohort {0 | 1} | floor {floor} | orphan {orphan} | maxdistance {maxdistance} | minclock {minclock} | minsane {minsane} ]"
