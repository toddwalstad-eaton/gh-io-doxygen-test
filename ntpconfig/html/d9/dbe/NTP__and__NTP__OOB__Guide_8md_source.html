1 # NTP Daemon (ntpd)
2 PX-Red uses a customized version of the NTP Daemon software for syncing the system clock to remote time servers. The PX-Red customizations to the NTP codebase allows the NTP Daemon to report status and clock change events to the NTP Out-of-Band (OOB) Objects. These NTP status and clock change events will in turn trigger the toolkit Configsubscriber proccess to perform the script actions detailed below.
3 
4 ## How NTP Adjusts the System Clock
5 
6 NTP uses two methods to adjust the system clock:
7 * Clockstep - If the system time is offset from the correct time by more than 1 second, NTP will step the system clock to within 1 second of the correct time. When a clockstep operation is performed, the Eaton custom code will update the `ClockAdjusted` OOB as well.
8 * Slew - When the system time is within 1 second of being correct, NTP will incrementally slew the clock to the correct time. Because clock slewing is done in time critical functions, there is no custom code in these functions.
9 
10 ## Configsubscriber Initialization and Action Scripts for NTP
11 
12 The [configsubscriber.json](../../rootfs/skeleton/etc/configsubscriber/configsubscriber.json) initialization file sets up the following change event actions on Out-of-Band (OOB) object updates:
13 1. If the `DateTime` OOB is changed, then the [`datetimeset.sh`](../../rootfs/skeleton/sbin/datetimeset.sh) and [`ntp_reboot`](./rootfs/etc/configsubscriber/ntp_reboot) scripts are called.
14 2. If the `ClockAdjusted` OOB is written to, then the [`clockadjusted`](../../rootfs/skeleton/etc/configsubscriber/clockadjusted) script is called.

22 ## NTP Daemon Startup Process
23 
24 The [NTP init script](./rootfs/etc/init.d/ntpd) will:
25 1. Read NTP configuration from the **DateTime** OOB.
26 2. Write the configured NTP servers and access restrictions to the [NTP configuration file `ntp.conf`](./rootfs/etc/ntp.conf) using the [`ntpconfig` utility](ntpconfig.md). The default target filesystem location of the `ntp.conf` file is:
27 The default NTP configuration file system location is:
28 > /etc/ntp.conf
29 3. Start the `ntpd` daemon.
30 
31 ### Best Practices for NTP Access Restrictions
32 
33 The access restriction settings written to the NTP configuration file are based on recommentations from the Network Time Foundation wiki [Access Restrictions](http://support.ntp.org/bin/view/Support/AccessRestrictions). See the [NTP init script](./rootfs/etc/init.d/ntpd) for more access restriction details.

35 # NTP Daemon Configuration and Operation Out-of-Band (OOB) Objects
36 
37 The following NTP OOB objects support the configuration and operation of the NTP daemon `ntpd`:
38 * [DateTime](../../documents/DateTime_OOB_Objects.md) - NTP server configuration
39 * ClockAdjusted - Trigger OOB for NTP clockstep.
40 * NtpConfig - NTP operational mode configuration.
41 * NtpStatus - NTP status.
42 
43 ## ClockAdjusted OOB
44 
45 The NTP daemon writes "1" to this OOB on every clockstep. This OOB enables registering for system data/time changes with the Configsubscriber process.
46 * [Default initialization for ClockAdjusted OOB](../../rootfs/skeleton/etc/cds/generic/97_datetime_defaults.json).
47 * [Default subscriptions for the Configsubscriber process](../../rootfs/skeleton/etc/configsubscriber/configsubscriber.json).
48 * [Default script the Configsubscriber runs when data/time changes occur](../../rootfs/skeleton/etc/configsubscriber/clockadjusted).
