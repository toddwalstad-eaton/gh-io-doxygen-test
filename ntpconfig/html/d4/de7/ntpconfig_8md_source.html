1 # `ntpconfig` program
2 The standard NTPD daemon does not save command-line configuration options to the NTP configuration file (ntp.conf).
3 The Eaton `ntpconfig` NTP configuration utility saves a select list of command-line options to the NTP configuration file.
4 
5 ## NTP Config Utility - Command-Line Rules for Saving Options to the NTP Config File
6 
7 1. Invoking any of the defined **Save Command-Line Options** will update the configuration file.
8 
9 2. Option parameters MUST be bracketed by double quotes (").
10 > OK: **ntpconfig -restrict "128.4.0.0 mask 255.255.0.0 #Allow DCnet clients"**
11 >
12 > NOT OK: **ntpconfig -restrict 128.4.0.0 mask 255.255.0.0 #Allow DCnet clients**
13 >
14 > Note that the '#Allow DCnet clients' comment text will also be added to
15 > the configuration file.
16 
17 3. An option parameter may be invoked multiple times.
18 > For example, to add a pool of servers to the `ntp.conf` file, the following command-line format could be used:
19 > > **ntpconfig -S "0.us.pool.ntp.org" -S "1.us.pool.ntp.org" -S "2.us.pool.ntp.org"**
20 
21 4. Invoking an option will delete all instances of that option from the current configuration file.
22 > For example, if the current `ntp.conf` file has the following "restrict" options:
23 > > **restrict default noserve**
24 > > **restrict 127.0.0.1**
25 > > **restrict 128.4.0.0 mask 255.255.0.0**
26 >
27 > Then after invoking the following "restrict" command-line option,
28 > > **ntpconfig -restrict "128.175.0.0"**
29 >
30 > The new `ntp.conf` file would contain only one "restrict" option:
31 > > **restrict 128.175.0.0**
32 
33 5. Invoking an option with no parameters will delete all instances of that option from the current configuration file.
34 > For example, if the current `ntp.conf` file has the following "restrict" options:
35 > > **restrict default noserve**
36 > > **restrict 127.0.0.1**
37 > > **restrict 128.4.0.0 mask 255.255.0.0**
38 >
39 > Then after invoking the following "restrict" command-line option,
40 > > **ntpconfig -restrict**
41 >
42 > The new `ntp.conf` file will have no "restrict" options.
