Detailed Description

Represents global or per-user password-management policy, such as expiration and complexity requirements.

For all members, unset = the system default (so we can tell for per-user use when to override the defaults)

Member Data Documentation

boost::optional<uint8_t> px::CUserPolicy::password_reuse_block_depth

Number of previous passwords to compare against and disallow.

0 is treated as 1 (changing to the same password is always blocked). Maximum is 32.

boost::optional<uint32_t> px::CUserPolicy::password_expiry_grace_days

Grace period for password expiration, in days.

Login is allowed during this time (though password must be changed before any more logins are allowed).

boost::optional<uint32_t> px::CUserPolicy::max_concurrent_sessions

Maximum concurrent login sessions a user may have.

0 or unset = unlimited

boost::optional<uint32_t> px::CUserPolicy::max_failed_attempts

Number of login failed attempts to trigger a lockout.

Note: Currently only used when provided as global policy rather than per-user policy

boost::optional<uint32_t> px::CUserPolicy::failed_login_attempt_window_secs

Number of seconds after which failed attempts are reset.

Note: Currently only used when provided as global policy rather than per-user policy

boost::optional<uint32_t> px::CUserPolicy::failed_login_wait_secs

Number of seconds to lockout if we get max_failed_attempts within the failed_login_attempt_window.

Note: Currently only used when provided as global policy rather than per-user policy
