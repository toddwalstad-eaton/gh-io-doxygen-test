Detailed Description

px::cds contains the CDS API objects - these are passed to and from the CDS using px::cds::CClient::submit() and related functions, to request that the Core Data Store do stuff.

Typedef Documentation

using px::cds::response_callback_t = typedef std::function<void ( std::shared_ptr<CRequestResponse> )>

The typedef of the standard CDS response callback function.

The callback function has one argument, a shared pointer to a (derivative of) the CRequestResponse class, containing the response information. The callback has no return (void).

typedef uint16_t px::cds::CallbackHandle

Assigned by CDS when callbacks are registered, this identifies where to send responses.

enum px::cds::RequestStatus

Status values.

Enumerator
INCOMPLETE 

Request has not finished processing.

SUCCEEDED 

Finished, good.

FAILED 

Finished, bad (some kind of Failure)

QUEUE_FULL 

Queue too full to fit response. Might work later.

RESPONSE_TOO_BIG 

Response is itself too large to fit in an empty queue, it will never succeed.

TIMEOUT 

Expiration time has passed.

PERMISSION_DENIED 

Client session or user not permitted to do this.

BAD_PARAMETER 

Fails due to invalid, missing, or bad parameter.

EMPTY_RESULT 

Failed with no results to return, but there should have been.

Normally because input parameters are entirely missing.

NOT_ABLE_TO_EXECUTE 

Failed because system or device was too busy or was unable to execute the request for some other reason.

strong

Whether to set or clear the given states.

Since no one process is responsible for setting all the Device State bits, probably the first two operations will be the only ones used in practice.

Enumerator

Turn ON specified state flags, leaving others alone.

op_clear_flags 

Turn OFF specified state flags, leaving others alone.

op_set_all_flags 

Set device state to exact value given.

enum px::cds::persistence
strong

Describes the persistence-to-disk behavior we want from a set operation.

Enumerator
save_always 

Unconditionally save to persistent storage.

save_if_saved_before 

Save if data for this tag is already persisted.

This is the default.

do_not_save 

Do not persist this data, whether this tag is already persisted or not.

clear_saved 

Erase any persistent copy of this tag's data (but keep in memory)
