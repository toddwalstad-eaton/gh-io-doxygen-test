Member Data Documentation

CallbackHandle px::cds::CRequestResponse::callback_handle

Which client-side callback do we call when response is ready? Set to one of the special values, or a value returned by px::cds::CClient::register_callback().

Default is CALLBACK_NONE.

uint32_t px::cds::CRequestResponse::transaction_id

Set by CDS to the same value returned by px::cds::CClient::submit().

Can be used by client to disambiguate responses.

px::string px::cds::CRequestResponse::session_token

Opaque string token to identify with which login session this request is associated.

Leave blank or set to the session_token returned in the response to the relevant px::cds::CLogin request. If left blank, CDS authorizes this request with the permissions (if any) associated with the client's Linux user ID.

px::CMonoTime px::cds::CRequestResponse::expiration_time

Cancel this request, returning error RequestStatus::TIMEOUT if not complete by this time.

If the client won't care about the response if it comes after some certain time (e.g. a data server will return a timeout to its client after this time, so no need for CDS to respond after then). Default = never.
