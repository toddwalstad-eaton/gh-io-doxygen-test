Detailed Description

Contains a command/control's both meta and instance information.

Member Data Documentation

px::string px::CDeviceCommand::name

The canonical name of the current command, e.g.

cSARstEnergy. Must be unique per device.

px::string px::CDeviceCommand::display_name

The display name of the current command.

e.g. Reset Energy

px::string px::CDeviceCommand::description

The description of the current command.

e.g. Reset all energy channels.

std::vector<px::CParameterInfo> px::CDeviceCommand::parameters

Represents a list of parameters (both input and output).

It may contain the parameter values if used during the execution.

std::vector<px::string> px::CDeviceCommand::associated_channel_canonical_names

Indicates the associated (affected) channel's MCL name.

Default/null indicates that no MCL channel is associated. notes: cannot use id because otherwise it requires binding to a particular device. This list gives UI hints to show the related channels.

bool px::CDeviceCommand::is_custom

Indicates that this is not a standard CCL command.

By default, the commands are standard.

bool px::CDeviceCommand::is_automatic

Indicates that this is an automatic command, e.g.

waveform capture, or time-sync. By default, the commands are non-automatic.

px::id_t px::CDeviceCommand::device_id

Fields below are instance fields, which should only be used for a particular command instance during the execution.

Represents the target device id.

px::CBitset<px::permissions> px::CDeviceCommand::permissions

Contains a list of Permissions (formerly called "roles") which can execute this command.

Example Permissions: px::permissions::control_basic - For resets and the like. px::permissions::control_oper - For breaker-open, px::permissions::setpoints - For things only needed at setup (setpoints, CT ratios, etc.)

bool px::CDeviceCommand::is_hidden

Indicates if this command should not be visible to the end user (UI).

The hidden command can be used internally or machine-to-machine, but should not be exposed to human users.
