enum px::sub::subscription_type : uint8_t
strong

Subscription types.

Defines the possible types of subscriptions that a client can use.

Enumerator
value_set 

Update to a channel's current/real-time value whether the value changes or not.

This is the most basic subscription type, and also the one that's the most performance-critical.

value_update 

Update to a channel's current/real-time value that changes the value.

Subscribers will not be notified if the value equals its previous value. A common case is when a device poll only updates a channels timestamp, no notification will be sent to the subscribers.

channel_settings 

Update to channel-level settings.

Channel settings are updated if (for example) a user or application alters the display name of a particular channel.

device_settings 

Update to device-level settings.

Device settings are updated if (for example) a user or application alters the display name of a particular device.

oob_data 

Update to out-of-band data.

OOB data is updated through an explicit API call for any number of reasons; this subscription type is used for all OOB data events, including deletions.

channel_state

device_state_recursive 

Update to all device state changes including those of any current and future sub-devices.

alarm 

Update to an alarm.

Alarms are updated when a change occurs that breaches a set threshold.

alarm_recursive 

Update to all alarms and sub alarms.

Similar to an alarm subscription except that notifications are sent for all sub device and channels as well.

digital_trends 

Update to a digital trend.

digital_trends_recursive 

Update to a digital trend included that of all sub devices/ channels.

change_signature 

Update to the change signature.

Change signature updates are recursive by nature meaning that any update to a sub change signature will propagate upwards.

device_create_recursive 

Update is sent whenever a device is created.

includes all sub devices

device_delete_recursive 

Update is sent whenever a device is deleted.

includes all sub devices

value_state_update 

Update to a channel's current/real-time value and state if either value or state has changed.

Subscribers will not be notified if the value and state equal to their previous value and state. A common case is when a Mercury subscriber subscribes to the channel for its value and state updates.

device_command 

Device's command, such as ResetEnergy or SetChannelValue.
