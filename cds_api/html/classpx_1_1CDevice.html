Detailed Description

Represents a device, which can be either a publisher, a device connected to the publisher, or a network connector such as INCOM port 1.

Devices contain channels and possibly other devices. Every system has at least one device, the "publisher" at ID 1, which has no parent.

Member Data Documentation

uint64_t px::CDevice::change_signature

Represents a change signature, to help avoid any unnecessary re-discovery by indicating if any significant change has occurred in this device branch.

This field shall not change if the only changes are the device's family, model, physical address, serial, vendor, or asset_id.

Changes to fields that trigger a change to change_signature will change this device's parent's change signature, and so on up to the root of the tree.

Default/null means that the device's family is unknown. Unused by CDS but may be used by device handlers and UI.

The physical network address of this device, relative to its parent device, such as port number, or INCOM address.

The publisher's network address should be null. Unused by CDS (except for logging) but may be used by device handlers and UI.

Default/null means unknown or none.

px::string px::CDevice::asset_id

An arbitrary customer-assigned Asset Identification tag for this device, used to track physical hardware.

Unused by CDS but may be used by device handlers and UI.

Default/null means that the Asset Id is unknown.

std::vector<unsigned char> px::CDevice::guid

A unique identifier for this device, which distinguishes it from ALL other devices, globally.

This field is not required for sub-devices, but it IS required for a publisher, since a publisher needs to be uniquely identified in a hosted solution.

Unused by CDS but used in other parts of the system (like iot_agent)

px::string px::CDevice::location

The physical location of this device, as assigned by the customer.

This is a completely free-form string, e.g. "Tom's office in CPO".

Unused by CDS but may be used by device handlers / UI Default/null means that the location is unknown.

px::string px::CDevice::contact

Administrative contact information for this device.

This is assigned by the customer and can take whatever form is most useful, e.g. "Tom x1234" or "bobjones@gmail.com".

Unused by CDS but may be used by device handlers / UI Default/null means that the administrative contact is unknown.

px::string px::CDevice::software_version

Software / firmware version for this device.

Unused by CDS but may be used by device handlers / UI The publisher's software_version is set to the Toolkit firmware version.

bool px::CDevice::is_submaster

Indicates if this device can serve as a submaster (parent) to child devices which are addressed through it.

Eg, the INCOM BIM-II. Does not apply to Ports.
