Detailed Description

Encapsulates a channel's configuration, as seen in the Mercury message "Channel".

See Core.proto for more details.

Enum Represents the primitive value type of the channel's measurement.

Default of 255 means the datatype is invalid or not applied.

uint32_t px::CChannelSettings::mcl_tag

The unique ID for this channel in the Master or Shared Channel List.

"mcl_tag" maps to the 'channel_id' attribute in the Channel protobuf object (see Core.proto), and was renamed in this class to avoid confusion with the CDS channel id property.

This numerical assignment is the "tag" for the channel in the PX White system and IoT Device Profiles.

PUBLIC px::string px::CChannelSettings::get_best_name ( size_t  max_len = 0) const

Get the best text for the Channel name that fits within the given length constraint (if any).

If length is > 0, will return a name that is <= that length, choosing the abbreviated or short names if they fit best within the length. If there is no length constraint, will choose the first of these that is available: (display) name, short_name, canonical_name, and finally abbreviated name. If none of these exist, will make a name "ChXXX" where XXX is the Channel ID.

Parameters
max_lenIf non-zero, the name returned must be <= this length.
PUBLIC float px::CChannelSettings::get_units_scaling ( ) const

Checks the units for this Channel and determines any scale factor necessary to convert the Channel's value to whole SI units.

Eg, if a Channel has units of "kilowatthours", then this function will return a scale factor of 1000.0 that can be multiplied against the Channel's value to get a reading in watthours. Checks for units starting with "kilo" (1000), "mega" (1000000), "milli" (0.001), "centi" (0.01), and "micro" (0.000001) (Others are possible, if called for later). In the case of "no units" or anything else, will return a scale factor of 1.0

Returns
A scale factor that can be applied against the Channel Value to convert it to whole SI units. Default return value is 1.0.
