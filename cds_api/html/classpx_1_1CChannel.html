Detailed Description

Channel as referred to by the CDS.

Through composition, brings all the pieces of a channel into one object, as in the px::proto::MclEntry protobuf object.

Note that the ID that identifies the Channel is actually stored in the ChannelSettings.

PUBLIC bool px::CChannel::get_field_by_name ( const px::string &  field_string,
px::CValue &  field_value 
) const

Get the passed field for this device by name (string).

This is only supported for public fields.

Unsupported fields (return false) alarm_settings.* state timestamp alarm_state trend_accumulator.*

Noted fields alarm_settings - without a "." accessor for a field under it, this returns false. channel_settings - without a "." accessor for a field under it, this returns false. trend_accumulator - without a "." accessor for a field under it, this returns false. channel_settings.multistate_map - Returns the label mapped value of "value" field by default. Will return another label mapped value if passed, e.g. channel_settings.multistate_map.1. channel_settings.* - see channel_settings.hpp for description
