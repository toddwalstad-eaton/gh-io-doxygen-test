Detailed Description

Represents a single alarm update, as used in the Mercury protocol.

Member Data Documentation

uint32_t px::CAlarmUpdate::index

Sequence of this update for this alarm ID, starting from 0.

Sequence IDs are never skipped.

uint64_t px::CAlarmUpdate::global_count

Global count (of alarm updates) corresponding to this update.

Global counts may be skipped but never decrease.

px::string px::CAlarmUpdate::str_val

String representation of the trigger value - particularly useful for multistates.

It's either a simple conversion or a multistate_map lookup from the Channel. Since it's derived, it's not to be persisted. It's always populated during alarm publishing and for alarm queries only when CGetAlarms.populate_str_val is specified.
