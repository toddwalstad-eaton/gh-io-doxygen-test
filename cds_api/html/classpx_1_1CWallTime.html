Detailed Description

Wall-clock time.

This is a particular kind of timestamp, known to be derived from the normal (wall/realtime) clock. It is functionally the same as a CTimestamp object for the most part, but making it its own type serves both to make it clear in code what kind of timestamp we're dealing with, as well as prevent accidental comparisons between monotonic and wall-clock times.

Since our normal realtime clock timebase is UTC, the CWallTime values are normally UTC as well. (So WallTime is NOT the time you would see on a clock the Wall, except in the Greenwich timezone.)

Factory function to create a new CWallTime instance set to "now".

This function uses the current system clock time (at the point at which it is invoked) to create a timestamp object. Normally our systems are set to keep UTC time, so then this value will be UTC.
