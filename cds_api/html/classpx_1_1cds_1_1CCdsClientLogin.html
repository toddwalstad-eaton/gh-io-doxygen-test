Detailed Description

Login with the credentials assigned to this process by virtue of its Linux user ID (assigned by "uid_perms" CDS configuration files).

This is useful for internal processes for which there is no user name or password.

Client provides the name of the requested "service" for login, (currently used only for logging purposes), and the desired timeouts. CDS fills in a list of authenticated permissions, and CRequestResponse::session_token, if login is successful.

Requests with blank CRequestResponse::session_token from this process get the same permissions as requests with the token returned from this; this API is for when you want to authorize another process with your Linux permissions (by passing it the token returned from this).
